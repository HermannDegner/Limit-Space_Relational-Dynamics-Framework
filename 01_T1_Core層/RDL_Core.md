# RDL_Core
*関係力学言語 中核数理層*

---

## ■ 0. 位置づけ

本文書は、関係力学言語（RDL: Relational Dynamics Language）の**中核数理層**を独立化したものである。

### 0.1 Core が扱う範囲

Core は以下を扱う：

```text
・基底量の定義（ξ / B / M_B / EFP / F / E）
・基底量間の基本式
・関係保存則
・ξ の残存性
・整合と跳躍の同一性
```

Core は以下を扱わない（外部参照）：

```text
・B 仮設の操作論              → SILN 分解（数理フレームワーク v2.2）
・B/M_B の耐久検査・残存・移植 → 構造破壊学（前提文書 v0.4）
・他理論の借用                 → 借用モジュール群（NAM / GR-MOD / FluctMOD）
・モジュール切替の判定         → RDL_切替判定 v1.0
・運用上の道具化リスク         → RDL 道具特性 v0.1
```

### 0.2 設計方針

Core は、RDL の**最小ダイナミクスと基底量**に絞る。
追従運用・破壊検査・借用判断は外部に置く。

```text
Core が扱う      ：基底量・基本式・保存則・最小ダイナミクスの骨格
Core が扱わない  ：追従運用・耐久検査・借用判断・切替判定・運用論
```

理由：

```text
・Core は最も汎用な再利用部品である
・自己検査機構を内蔵すると軽量用途で過剰になる
  （例：NPC的AIエージェントは薄い構造破壊で十分）
・用途に応じて検査の重さを選べる構造が望ましい
```

Core = 動かす部品。構造破壊学 = 検査する部品。両者の分離が本文書の前提である。

## ■ 1. 哲学的基底（ξ + B + M_B）

RDL の哲学的基底は三項組である：

```text
ξ   ：残存・揺らぎ・根源的自由度
B   ：境界（観測者が引いた境界の組）
M_B ：境界 B における整合慣性
```

これらは独立した量ではない。相互依存する三項組として基底を成す。

### 1.1 ξ（残存・揺らぎ）

ξ は確率的変動・根源的自由度を表す。
ξ は B に対して残存する：B を引いても ξ は M_B の中・縁・外に残り続ける。

```text
ξ は基底の「閉じなさ」を表現する量
B をどう引いても、M_B をどう仮設しても、ξ は染み込んでいる
```

### 1.2 B（境界）

B は観測者が引いた境界の組である：

```text
B = { 空間スケール, 時間スケール, 評価関数, 用途, 観測者 }
```

B は与件ではなく**選択**である。

```text
・B は観測者が選ぶ
・観測者自身も M_B である（観測者 = M_{B_observer}）
・したがって B の選択は観測者の M_B から派生する
・純粋に外側から B を引く観測者は存在しない
```

これは限界空間の構造的非対称：「境界を引く」行為だけは irreducible である。

### 1.3 M_B（整合慣性）

M_B は境界 B における整合慣性場である。

```text
M_B = P_B(整合慣性場)
```

M_B は**仮設**である。直接観測できない。
行動・出力・反応のパターンから推定した仮説に過ぎない。

独立した「M そのもの」は存在しない。すべての M は B 依存の M_B である。

### 1.4 M_B の入れ子ネットワーク性

M_B は孤立した単位ではない。

任意の M_B の内部には、より小さい境界 B_i によって仮設された複数の M_Bi が存在しうる。
また、その M_B 自身も、より大きな境界 B_parent における上位 M_B の一部として機能しうる。

```text
M_B(parent)
  ├─ M_B1
  │   ├─ M_B1-a
  │   └─ M_B1-b
  ├─ M_B2
  └─ M_B3
        └─ M_B3-a
```

したがって M_B は**入れ子状に接続された M_B ネットワーク**として存在する。

> **M_B 入れ子ネットワーク原理**：M_B は単独の整合慣性ではなく、複数の下位 M_B を内包し、上位 M_B に内包される、入れ子状ネットワークとして存在する。

この構造により：

```text
・局所破断 → 下位 M_B の再編のみ（上位 M_B は維持される場合がある）
・広域破断 → 上位 M_B の再編（下位 M_B はそれに従って再配置される）
・連鎖破断 → 下位 M_B の誤差蓄積が上位 M_B を揺らし、M_B ネットワーク全体が相転移する
```

RDL における「構造」は単一の安定点ではなく、**境界ごとに仮設された整合慣性の多層ネットワーク**として記述される。これが T2 階層構造モジュールの Core 的根拠であり、ラベル M_B ネットワークの入れ子性の基礎でもある。

## ■ 2. 実用的基底（EFP / F / E / M_B / ξ）

数理運用において扱う基底量は五項である：

```text
EFP : 素流圧（Elementary Flux Pressure）
F   : 作用解釈
E   : 慣性誤差
M_B : 整合慣性（哲学的基底から継承）
ξ   : 残存・揺らぎ（哲学的基底から継承）
```

ただし運用上、E の蓄積量として **H（熱）を補助的に導入する**（2.4 参照）。
H は基底量ではなく、E から派生する補助量である。

※ v1.1 以前に存在した S（状態）は除去。M_B の読み出し断面として M_B に吸収される。

### 2.1 EFP（素流圧）

未解釈の作用束。M_B の外側から流入する。

**EFP は直接観測できない。**

観測者が実際に持つのは F だけである。
EFP は F から M_B を使って逆算した推定概念に過ぎない。

```text
EFP_estimated = interp⁻¹(M_B, F)
```

𝒦 と同じ位置づけ：到達不能だが操作上有用なため仮設置する。

### 2.2 F（作用解釈）

M_B が EFP を解釈した結果。
観測者が実際に持つ唯一の作用情報。

```text
F = interp(M_B, EFP)
```

性質：

```text
・知覚の質は M_B に依存する
・M_B がリッチなほど EFP から多くを読み取れる
・M_B がフラットなら EFP はほぼ素通り
```

### 2.3 E（慣性誤差）

```text
E(t) = F(t+Δ) - M_B·F(t)
```

- `M_B·F(t)`：M_B の慣性が F(t) に乗ったまま次へ流れた値（**慣性投影**）
- `F(t+Δ)`：実際に来た次の F
- `E(t)`：慣性投影と実際の F の差 = 慣性誤差

**Δ（時間差）の意味：**

```text
Δ = 0 は完全確定 = 限界空間で扱えない
Δ > 0 が必然

Δ 小  ：因果が明確・M_B 更新しやすい
Δ 大  ：因果が曖昧・M_B 更新しにくい
Δ ∞  ：因果不明・M_B は更新されない
```

### 2.4 H（熱・誤差蓄積：派生量）

```text
H     = ||H_vec||
H_vec ≡ δM_B(t)
```

慣性誤差の蓄積。H は E の時系列積分として蓄積する。
H_vec は M_B の未拘束自由度における振動。

H は基底量ではなく、E から派生する補助量である。
ただし跳躍条件の判定に直接関わるため、基底と並んで明示する。

## ■ 3. 基本式（Base Model）

Core の**連続更新ダイナミクスは三式で閉じる**。
ただし跳躍判定には派生量 H と実効閾値 θ_eff を用いる（第6章参照）。

```text
F(t)    = interp(M_B, EFP)         // ① EFP を M_B が解釈
E(t)    = F(t+Δ) - M_B·F(t)        // ② 慣性投影との差が誤差
dM_B/dt = f(M_B, E, ξ)              // ③ M_B は誤差と揺らぎで更新される
```

### 3.1 ダイナミクスの流れ

```text
EFP（仮設・直接観測不能）
    ↓ interp(M_B, EFP)
F(t)（作用解釈・観測者が持つ唯一の情報）
    ↓ M_B·F(t) 慣性投影
    ↓ F(t+Δ) が実際に来る
E(t) = F(t+Δ) - M_B·F(t)（慣性誤差）
    ↓
H 蓄積 → 跳躍（M_B → M_Δ → M_B'）
```

### 3.2 M_B の固有分解

```text
M_B = V_B Λ_B V_B⁻¹
```

- V_B：安定方向（境界 B における慣性の向き）
- Λ_B：強度（各方向の慣性の強さ）

### 3.3 M_B の相状態

M_B は単一の固定構造ではなく、相状態を持つ：

```text
M_act  : 活性断面（現在強く機能している）
M_lat  : 潜在断面（まだ発現していない）
M_Δ    : 再編相（変化の途中）
M_B'   : 再編後の M_B
```

これらは別の M ではなく、同じ M_B の異なる相状態である。

## ■ 4. 関係保存則

Core 中核の保存則：

```text
||M_B|| · D[ξ] = 𝒦
```

ここで：

```text
||M_B|| : 整合慣性のノルム（慣性の強さ）
D[ξ]    : ξ の拡散・残存度
𝒦      : 関係保存量（境界 B における不変量）
```

### 4.1 保存則の意味

慣性が強いほど、揺らぎは拡散しにくい。
慣性が弱いほど、揺らぎは大きく拡散する。
両者の積は B における不変量 𝒦 として保たれる。

```text
||M_B|| 大 ⇄ D[ξ] 小  （堅い構造・少ない揺らぎ）
||M_B|| 小 ⇄ D[ξ] 大  （柔らかい構造・大きな揺らぎ）
```

### 4.2 𝒦 の位置づけ

𝒦 は EFP と同様、**到達不能だが操作上有用なため仮設置される量**である。

```text
𝒦 は B に依存する
𝒦 は直接観測できない
𝒦 は ||M_B|| と D[ξ] の積として逆算される
```

### 4.3 「保存」の意味（B 内保存近似）

本式は**絶対保存則ではない**。

```text
𝒦 は B に依存する仮設量である
B が変われば 𝒦 も変わる
B 自体が動性を持つため、𝒦 も静止しない
```

したがって本式は、**境界 B において十分安定に見なせる関係保存近似**である。

```text
B 固定の局所近似として        : 保存則として扱える
B 越境・B 動性を含む文脈で    : 近似が破断する
```

「保存則」という名称は、B 内で十分安定な近似として運用する場合の通称である。
B の動性を扱う文脈では、本式は局所的な関係制約として再解釈される必要がある。

## ■ 5. ξ の残存性

関係保存則の直接の帰結として、ξ の残存性が導かれる：

```text
D[ξ] = 𝒦 / ||M_B|| ≠ 0
```

||M_B|| が有限である限り、D[ξ] はゼロにならない。

### 5.1 残存性の意味

```text
・B を引いても ξ は M_B の中・縁・外に残る
・SILN 分解で M_B を仮設しても、その M_B には常に ξ が染み込んでいる
・分解結果が「動的」であるのはこのため
```

### 5.2 残存性と動性の対構造

ξ の残存性は、B の動性と対構造をなす：

```text
ξ 残存性 ：B を引いても ξ は残る（B の不徹底性）
B の動性 ：引いた後も B は静止しない（B の非静止性）
```

B は二重に閉じない。
したがって Core 上に構築される SILN 分解も二重に閉じない（→ 追従運用化）。

## ■ 6. 整合と跳躍の同一性

Core における最も非自明な命題：

> **整合と跳躍は同一の操作の異なる相である。**

### 6.1 整合（M_B の維持）

通常時、M_B は F(t) を慣性投影し、E(t) を吸収しながら自己を維持する：

```text
H(t) < θ_eff(t) の領域では
  dM_B/dt は M_B 自身の安定方向 V_B に沿った微小更新に留まる
  → 「整合」モード
```

### 6.2 跳躍（M_B の再編）

H が実効閾値 θ_eff を超えると、M_B は相状態 M_Δ を経由して M_B' に再編される：

```text
H(t) > θ_eff(t) の領域では
  dM_B/dt は V_B の外へ漏れる
  M_B → M_Δ → M_B' （再編）
  → 「跳躍」モード
```

ここで：

```text
θ_eff(t) = θ + g(ξ(t))

θ        : 跳躍閾値（B 依存の構造量）
g(ξ)     : ξ が閾値を揺らす関数
```

ξ は閾値に直接加算されるのではなく、**閾値を揺らす関数 g を介して**作用する。
これにより跳躍境界そのものが ξ の影響で揺れる。

### 6.3 同一性の主張

整合と跳躍は別個の操作ではない。**同じ dM_B/dt = f(M_B, E, ξ) の異なる領域における顕現**である。

```text
整合 = 跳躍の蓄積前段階
跳躍 = 整合の累積誤差が実効閾値を超えた状態

両者の境界は θ_eff(t) という連続パラメータが画定する
両者は連続的に接続している
```

θ_eff そのものが g(ξ) を介して ξ の影響を受けるため、整合と跳躍の境界自体が揺れる。

### 6.4 同一性の含意

```text
・「安定」と「変化」は対立概念ではない
・「秩序」と「混沌」は対立概念ではない
・両者は同一ダイナミクスの異なる相にすぎない
・したがって M_B の動性は欠陥ではなく構造的帰結である
```

Core は構造破壊学の対象を提供する。構造破壊学は Core の運用に証明書を付ける。

判定は Core にも借用モジュールにも置かない（循環参照を避けるため）。
