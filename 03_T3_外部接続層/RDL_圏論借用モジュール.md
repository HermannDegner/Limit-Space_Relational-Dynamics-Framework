## ■ 0. 位置づけ

圏論はRDLの中核数学として採用するのではなく、**関係構造の整合性検査器**として借用する。

```
圏論      = 関係構造が保たれているかを見る道具
構造破壊学 = その関係構造がどこで壊れるかを見る運用
```

圏論で「綺麗に書ける部分」を採用するのではなく、**綺麗に書こうとしたときに破綻する場所を見る**のが主な用途。

本文書はT3として圏論語彙をRDLに接続する。実際の検査操作はT2（圏論的構造破壊検査プロトコル）、AIコーディングへの応用はT4（AIコーディング用可換性テスト）が担う。

## ■ 1. 基本語彙の対応

### 1.1 対象（Object）

```
圏論  ：対象 A, B, C, ...
RDL  ：SILN（境界Bのもとで一時的に安定しているパターン）
```

圏論では対象の内部構造を直接展開せず、主に射との関係として扱うが、RDLのSILNは内部にξを持つ。完全な対応ではなく、**操作的近似**として使う。

### 1.2 射（Morphism）

```
圏論  ：射 f: A → B（対象間の矢印）
RDL  ：W_ij（SILN間の接続・依存・作用・データ流）
```

射の方向性はW_ijの方向性に対応する。

### 1.3 合成（Composition）

```
圏論  ：g ∘ f: A → C（射の連鎖）
RDL  ：操作の連鎖・SILNの変換パイプライン
```

合成が定義できない = 操作プロトコルの破断候補。

### 1.4 恒等射（Identity）

```
圏論  ：id_A: A → A（対象を変えない射）
RDL  ：「同じSILNとして扱う」というBの安定性
```

恒等射が怪しい = 「同じもの」として扱うBが不安定（境界設定の問題）。

### 1.5 可換図式（Commutative Diagram）

```
圏論  ：異なる経路を辿っても同じ対象に到達する
RDL  ：異なる操作経路を通っても同じ構造・結果に至る
```

```
A
↓ f        ↘ h
B → C      D
    ↑ g ∘ f    ↑ k ∘ h
```

```
A --f--> B --g--> C
|                 ^
h                 |
v                 k
D ---------------/
```

`g ∘ f = k ∘ h` が成り立つとき、この図式は可換である。
成り立たないとき = **経路依存のH（熱蓄積）または破断候補**。

### 1.6 関手（Functor）

```
圏論  ：圏の構造を保ちながら別の圏へ写す操作
RDL  ：翻訳接続系（外部理論↔RDL変数の変換）
```

関手で構造が保てない = 翻訳接続の破断。変換後にW_ijの方向や合成可能性が崩れていないか確認する。

### 1.7 自然変換（Natural Transformation）

```
圏論  ：二つの関手の間の構造を保つ写像
RDL  ：二つの翻訳方式（借用モジュール）が整合しているかの検査
```

自然変換が作れない = 二つの翻訳方式が整合しない。複数の借用モジュールを並用する際の衝突点として使える。

### 1.8 極限・余極限（Limit / Colimit）

```
圏論  ：複数の対象・射の「最適な統合点」
RDL  ：複数SILNの統合・貼り合わせ
```

```
極限が取れない  = 統合不能・共通構造がない
余極限が壊れる = 貼り合わせ時の境界衝突（B同士の干渉）
```

## ■ 2. 検査の基本操作

圏論を破壊検査に使う場合の基本手順：

```
1. RDL構造を圏論的に図式化する
     SILN → 対象
     W_ij → 射
     操作連鎖 → 合成

2. 可換性を問う
     「この二つの経路は同じ結果に至るか？」

3. 可換するなら → その範囲では構造が安定
   可換しないなら → 以下の候補として記録：
     ・経路依存のH（操作順序で結果が変わる）
     ・ξの影響（揺らぎが経路差を生んでいる）
     ・B設定ミス（境界が適切に引かれていない）
     ・W_ij破断（接続関係が実態と合っていない）
```

## ■ 3. RDL変数との対応表

| 圏論概念 | RDL変数 | 破断時の読み方 |
|---|---|---|
| 対象 | SILN | SILNのBが不安定 |
| 射 | W_ij | 接続・依存が成立しない |
| 合成 | 操作連鎖 | プロトコルの破断 |
| 恒等射 | B安定性 | 「同じもの」扱いのB設定ミス |
| 可換図式 | 経路整合性 | H蓄積・ξ残存・破断候補 |
| 関手 | 翻訳接続系 | 翻訳で構造が保てない |
| 自然変換 | 借用モジュール整合 | 複数翻訳方式の衝突 |
| 極限 | SILN統合 | 統合不能・共通構造なし |
| 余極限 | SILN貼り合わせ | 境界衝突 |

## ■ 4. ξとの関係

圏論の枠組みで消えない部分をξとして扱う。

```
圏論的に記述できた部分 → 構造として安定している領域
圏論的に記述できない部分 → ξ残存・B再仮設候補
```

圏論で「綺麗に書けない」ことはRDL的には失敗ではない。
書けない部分がそのままξの可視化になる。

## ■ 5. 有効範囲と限界

### 有効範囲

```
・複数の設計経路を比較して破断点を探す（可換性テスト）
・翻訳接続が構造を保っているか確認する（関手性検査）
・複数借用モジュールの整合性を確認する（自然変換検査）
・SILN統合・貼り合わせの境界衝突を発見する（極限検査）
```

### 限界（圏論をCore数学にしない理由）

```
・圏論では対象の内部構造を射・関係・保存構造を通じて扱う傾向が強い
  → RDLのSILNが内部に持つξは、射の関係だけでは保持しきれない

・圏論は構造の保存を記述する言語である
  → ξ残存性（構造が閉じない）はそもそも圏論の記述外

・圏論は動態も記述できるが、標準的な借用だけでは
  RDLのdM_B/dt・H蓄積・跳躍・ξ残存を十分には保持できない
  → 動態部分はRDL側の記述として残す
```

圏論はRDLの部分的な整合性を照らすライトとして使う。全体を照らすことはできない。

## ■ 6. 圏論化してはいけないもの

以下は、無理に対象・射・図式へ変換しない。

```
・未確定の違和感
・まだBを引いていない直感
・設計者の判断保留
・非言語的な要求
・禁止補完領域
・破断条件そのもの
・観測者Bの揺れ
```

これらを圏論図式に押し込むと、ξが消えたように見える。
しかし実際には、図式外へ排出されただけである。

**圏論化できない部分は、失敗ではなくξ残存として記録する。**
