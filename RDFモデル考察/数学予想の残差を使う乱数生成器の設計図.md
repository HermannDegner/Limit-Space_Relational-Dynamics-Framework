# 数学予想の残差を使う乱数生成器の設計図

## 0. 目的

本設計図は、**数学的に強い法則性を持つ系列**そのものではなく、
そこから**こぼれ落ちる残差**を抽出し、
**乱数的に利用可能な擬似乱数生成器**として構成するための基本設計をまとめる。

ここでいう乱数は、暗号学的完全性を最初から目指すものではない。
むしろ、

- 強い法則
- その法則の有効範囲
- 法則で吸収しきれない微小差分
- 境界付近での揺らぎ

を利用した、**構造付き揺らぎ生成器**として扱う。

---

## 1. コア発想

乱数を「完全無秩序」としてではなく、

> 法則が説明しきれなかった残差

として扱う。

数学予想や強い数論的規則は、長い範囲で極めて強い整合性を見せることがある。
しかしその整合性は絶対ではなく、

- 近似である
- 局所的である
- 観測範囲依存である
- 破れや偏りを持ちうる

この**残り滓**を抽出し、混合し、整形することで、乱数的出力へ変換する。

---

## 2. 基本原理

### 2.1 残差定義

最小形は以下である。

```text
residue(n) = actual(n) - predicted(n)
```

ここで：

- `actual(n)` は実際の数列・累積値・観測量
- `predicted(n)` は予想・近似式・期待値・平均傾向

である。

---

### 2.2 剰余型残差

差分が巨大になりすぎる場合は、法則で割った余りとして扱える。

```text
residue_mod(n) = actual(n) mod rule(n)
```

これは値域圧縮に有効。

---

### 2.3 傾向除去型残差

長期トレンドを引いて微小揺らぎのみを取り出す。

```text
trend(n) = smooth(actual(1..n))
residue(n) = actual(n) - trend(n)
```

---

## 3. 数学源の候補

### 3.1 メビウス関数累積

```text
M(n) = Σ_{k=1..n} μ(k)
```

候補残差：

```text
r(n) = M(n) / sqrt(n)
```

または

```text
r(n) = M(n) - predicted_envelope(n)
```

特徴：

- 振れがある
- 偏りが完全には読みにくい
- 長距離構造を含む

---

### 3.2 素数間隔

```text
g(n) = p_{n+1} - p_n
```

候補残差：

```text
r(n) = g(n) - log(p_n)
```

特徴：

- 平均傾向はある
- 局所揺らぎが大きい
- 離散的で扱いやすい

---

### 3.3 素数計数関数の残差

```text
r(n) = π(n) - li(n)
```

特徴：

- 強い法則の差分
- 長期構造が濃い
- 低周波成分が強い可能性

---

### 3.4 約数関数・分割数・数論関数一般

例：

```text
r(n) = d(n) - trend_d(n)
r(n) = p(n) - asymptotic_p(n)
```

---

## 4. 生成器アーキテクチャ

全体構造は以下。

```text
数学系列
  ↓
予想値・近似値・包絡線
  ↓
残差抽出
  ↓
正規化
  ↓
混合
  ↓
非線形変換
  ↓
ビット化 / 数値化
  ↓
出力
```

---

## 5. モジュール分解

## 5.1 Source Module

役割：数学系列の供給

入力例：

- 整数 `n`
- 内部状態 `state`

出力例：

- `actual(n)`
- 補助系列

要求：

- 再現可能
- 実装可能
- 計算コストが制御可能

---

## 5.2 Predictor Module

役割：法則側の生成

出力：

```text
predicted(n)
```

候補：

- 解析的近似
- 移動平均
- 包絡線
- 漸近式
- 前段値からの予測

このモジュールが「何を法則とみなすか」を決める。

---

## 5.3 Residue Extractor

役割：残差抽出

基本式：

```text
r(n) = actual(n) - predicted(n)
```

代替：

```text
r(n) = frac(actual(n) / predicted(n))
r(n) = sign(actual(n) - predicted(n))
r(n) = (actual(n) XOR transformed(predicted(n)))
```

---

## 5.4 Normalizer

役割：値域圧縮と統計安定化

例：

```text
u(n) = tanh(α r(n))
u(n) = frac(β |r(n)|)
u(n) = (r(n) - μ_r) / σ_r
```

目的：

- 極端な値の抑制
- 値域固定
- 比較可能化

---

## 5.5 Mixer

役割：偏り緩和・周期崩し

方法：

```text
m(n) = hash(u1(n), u2(n-k), u3(n-ℓ))
```

または

```text
m(n) = frac(a*u1 + b*u2 + c*u3 + d*state)
```

混合候補：

- 異なる数学源
- 異なるスケールの残差
- 時間遅延
- 自己フィードバック

---

## 5.6 Nonlinear Transformer

役割：単純構造の露出を抑える

例：

```text
x_{n+1} = frac(m(n) + x_n^2 + γ)
```

または

```text
x_{n+1} = frac(sin(A*m(n) + B*x_n))
```

注意：

- 非線形化は有効だが、説明可能性は落ちる
- 強すぎると単なる一般的PRNGになり思想が薄れる

---

## 5.7 Output Module

出力形式：

### ビット出力

```text
bit(n) = 1 if x_n > τ else 0
```

### 整数出力

```text
rand32 = floor(x_n * 2^32)
```

### 浮動小数出力

```text
rand01 = frac(x_n)
```

---

## 6. 最小実装案

### 案A：メビウス残差PRNG

```text
M(n) = Σ μ(k)
r(n) = M(n) / sqrt(n)
u(n) = frac(abs(r(n)) * A)
x_{n+1} = frac(ν(n) + x_n * B)
```

出力：

```text
rand(n) = x_n
```

利点：

- 構造が明確
- 法則と残差の関係が見える

欠点：

- 計算負荷
- 偏りの可能性

---

### 案B：素数間隔残差PRNG

```text
g(n) = p_{n+1} - p_n
r(n) = g(n) - log(p_n)
u(n) = frac(abs(r(n)) * A)
x_{n+1} = frac(ν(n) + C*x_n + D)
```

利点：

- 実装しやすい
- 離散系列として扱いやすい

---

### 案C：多源混合型

```text
r1(n) = residue from Möbius
r2(n) = residue from prime gaps
r3(n) = residue from divisor function
m(n) = hash(r1(n), r2(n-3), r3(n-7), x_n)
x_{n+1} = frac(m(n))
```

利点：

- 偏り低減
- 構造の重ね合わせが可能

---

## 7. 重要な設計思想

### 7.1 乱数は無秩序ではない

この生成器は、無秩序を模倣するのではなく、

> 強い法則からこぼれた差分を再利用する

ことに意味がある。

---

### 7.2 残差は「法則の外」ではない

残差は法則が粗すぎて拾えなかった部分であり、
法則の反対物ではない。

---

### 7.3 これは構造付き乱数である

この生成器が出すのは、完全無記憶乱数よりも、

- 歴史を持ち
- 法則の痕跡を持ち
- 境界の名残を持つ

揺らぎである。

---

## 9. 注意点

### 9.1 暗号用途は別問題

この設計は原理的に決定論的であるため、
そのままでは暗号用乱数としては不十分である可能性が高い。

---

### 9.2 数学構造由来の偏りが残る

元の系列の性質が強く残る可能性がある。
そのため、混合や非線形変換は必須に近い。

---

### 9.3 計算コストの問題

メビウス関数や素数関連量はコストが高い場合がある。
用途に応じて近似・キャッシュ・前計算が必要。

---

## 10. 発展方向

### 10.1 境界検知型

「予想が破れそうな地点」や「変動が増える地点」を重点利用する。

```text
if |r(n)| > threshold:
    use strongly
else:
    use weakly
```

---

### 10.2 熱蓄積型

残差をそのまま使わず、蓄積熱として扱う。

```text
H(n+1) = λH(n) + |r(n)|
if H(n) > θ:
    jump / reseed / transform
```

これにより「安定→蓄積→破れ→再編」のダイナミクスを持たせられる。

---

### 10.3 多層構造型

- 低周波層：長期残差
- 中周波層：局所揺らぎ
- 高周波層：ビット化後の微細変動

を分けて扱う。

---

## 11. RDF的圧縮解釈

この生成器を一文で言えば：

> 法則そのものではなく、法則からこぼれ落ちた未整合残差を、構造付き揺らぎとして再利用する生成器

さらに圧縮すると：

> 乱数は、整合慣性が世界を説明しきれずに残した滓である

## 12. 最小仕様書

### 入力

- 初期状態 `seed_state`
- 整数ステップ `n`
- 数学源選択 `source_type`

### 出力

- `rand01` in [0,1)
- `rand32` in [0, 2^32)
- `bit`

### 内部状態

- 現在ステップ `n`
- 数学系列バッファ
- 残差履歴
- 混合状態 `x_n`
- 必要なら熱 `H_n`

---

## 13. 擬似コード全体像

```text
initialize state x, H, n
loop:
    actual      = source(n)
    predicted   = predictor(n)
    residue     = actual - predicted
    normalized  = normalize(residue)
    mixed       = mix(normalized, x, history)
    x           = nonlinear_transform(mixed, x)
    H           = heat_update(H, residue)
    if H > threshold:
        x = jump_reconfigure(x, history, residue)
        H = reset_or_decay(H)
    output frac(x)
    n = n + 1
```

---

## 14. まとめ

この設計図の主眼は、
乱数を単なるノイズ源としてではなく、

- 法則
- 近似
- 有効範囲
- 残差
- 境界
- 破れ

のダイナミクスから構成する点にある。

数学予想の残差は、
「証明されなかった真理の失敗」ではなく、
**法則性の解像度限界からこぼれた情報**として読める。

そのこぼれた部分を使うことで、
単純な一様乱数とは違う、
**意味を持った揺らぎ生成器**を設計できる。
