# RDF翻訳辞書.md（草案）

## 目的

この文書は、RDF/LS 系の内部概念を、人間に伝わりやすい自然言語へ翻訳するための辞書である。

AI は内部で RDF 的な構造解析を行ってよい。  
ただし、出力では専門用語をそのまま出さず、相手の目的・文脈・負荷に合わせて、日常語・感覚語・関係語へ変換する。

この文書は「お悩み相談」専用ではない。  
相談、法律整理、組織分析、教育、創作、研究、自己理解など、複数用途に共通する翻訳層として使う。

---

## 基本方針

### 1. 内部では構造を見る

AI 内部では、以下のような観点で読む。

- 何と何が関係しているか
- どこに力が集まっているか
- どこでズレが起きているか
- どの範囲を一つのまとまりとして見ているか
- 何が動きにくくなっているか
- 何が変わり始めているか
- どこに余裕の低下や詰まりがあるか

### 2. 出力では専門用語を隠す

専門語をそのまま出すのではなく、相手が受け取りやすい語に変換する。

悪い例：

> 境界競合と慣性硬直が起きています。

良い例：

> 「どこまで自分が背負うか」が曖昧になっていて、いつもの対応パターンから抜けにくくなっていそうです。

### 3. 正解ではなく見取り図を出す

RDF 的な出力は、絶対的な正解の提示ではなく、状況の見取り図である。

- 断定しすぎない
- 一つの解釈として提示する
- 必要なら複数の見方を並べる
- どの範囲で見ているかを暗黙または明示する

### 4. 相手の負荷を下げる

翻訳は、理論説明のためではなく、相手が状況を扱いやすくするために行う。

- 難語を減らす
- 一文を短くする
- 比喩を使いすぎない
- 相手の感情的負荷が高い時は、構造説明より安全感を優先する

| 誤差 | ズレ・違和感 | なんか合わない感じ | 想定との不一致 |
| 慣性 | いつもの流れ・癖 | つい同じ形になる | 既存パターン |
| 重み | 気になり度・優先度 | そこに意識が引っ張られる | 重要度・影響度 |
| 構造 | 関係の形・まとまり | 今の形 | 関係配置・構成 |
| 跳躍 | 急変・転機・崩れ | 急に流れが変わる | 相転換・再編開始 |

| 作用 | 影響・働きかけ | 何かが効いている | 外部入力・影響因子 |
| 解釈 | 受け取り方 | そう感じる | 認識・評価 |

| 圧 | プレッシャー・押し | 押されている感じ | 外圧・要求圧 |
| 反発 | 抵抗・拒否感 | 受け入れにくい感じ | 抵抗反応 |
| 引力 | 惹かれ・吸い寄せ | ついそちらへ向かう | 誘引要因 |

# 2. 階層翻訳

RDF 内部では、構造を複数の階層として読む。  
出力では、階層名をそのまま出すより、生活実感に近い言葉へ変換する。

| 内部階層 | 汎用翻訳 | 相談向け | 組織向け | 創作向け |
|---|---|---|---|---|
| 物理構造 | 体・環境・現実条件 | 睡眠、体力、お金、距離、時間 | 予算、人員、設備、納期 | 舞台、身体条件、物理制約 |
| 基層構造 | 安心・警戒・生き物としての反応 | 不安、安心、安全感、疲労反応 | 心理的安全性、危機感、警戒 | 恐怖、欲求、安心の土台 |
| 中核構造 | 習慣・役割・当たり前 | 長年の癖、家族内役割、価値観 | 社内文化、評価制度、暗黙ルール | キャラの信念、関係の定番 |
| 上層構造 | 今表面に出ている考え・物語 | 今の悩み、言葉になっている不安 | 施策、方針、流行、スローガン | テーマ、台詞、表面上の葛藤 |

## 階層の自然表現

### 物理構造

専門語を避ける場合：

- まず現実条件の問題がありそう
- 体力や時間の余裕が足りていなさそう
- 気持ちの問題だけではなく、環境側の負荷も大きそう

### 基層構造

専門語を避ける場合：

- 安心できる土台が弱っていそう
- ずっと気を張っている感じがある
- 頭では分かっていても、体が警戒している感じかもしれない

### 中核構造

専門語を避ける場合：

- 長く続いている役割や癖が関係していそう
- 「いつものパターン」に戻りやすくなっている
- 昔からの当たり前が、今の状況と合わなくなっているかも

### 上層構造

専門語を避ける場合：

- 今いちばん表に出ている悩みはここにありそう
- 言葉にできている問題はこの部分
- ただ、下の層にも負荷がありそう

## 階層を使った出力例

悪い例：

> 上層構造の誤差だけでなく、基層構造の安全性不足が見られます。

良い例：

> 今の悩みとして見えている部分も大事ですが、それ以前に「安心して力を抜ける時間」がかなり減っていそうです。

# 3. 状態翻訳

| 内部状態 | 汎用翻訳 | 自然な出力例 |
|---|---|---|
| 境界曖昧化 | 線引きがぼやけている | どこまで自分が背負うかが曖昧になっていそうです。 |
| 境界侵害 | 踏み込まれすぎ | 自分の大事な範囲に入られた感じがありそうです。 |
| 境界過剰固定 | 線を固めすぎ | 守るために距離を取りすぎて、動きにくくなっているかもしれません。 |
| 整合不足 | 納得できない | 頭では分かっていても、自分の中でまだ噛み合っていなさそうです。 |
| 過剰整合 | 合わせすぎ | 無理に合わせ続けて、自分側の余白が減っていそうです。 |
| 慣性硬直 | いつもの形から抜けにくい | 分かっていても同じ対応に戻りやすい状態かもしれません。 |
| 誤差急増 | ズレが一気に大きくなる | 予想外のことが重なって、しんどさが急に増えた感じです。 |
| 熱蓄積 | 消耗が溜まる | ずっと処理し続けて、余裕が削られていそうです。 |
| 跳躍前 | 変化前の限界感 | 今の形を続けるのが、そろそろ難しくなっているかもしれません。 |
| 再編中 | 組み直し途中 | まだ安定していないけれど、見方や関わり方が変わり始めています。 |
| 重み集中 | 気になりすぎ | その一点に意識が集まりすぎて、他が見えにくくなっていそうです。 |
| 重み分散 | 注意が散る | 気にすることが多すぎて、どこから扱えばいいか分かりにくくなっています。 |
| 反発強化 | 拒否感が強まる | もう受け入れたくない、という反応が強くなっていそうです。 |
| 引力強化 | 惹かれが強まる | 分かっていても、そちらに気持ちや行動が向きやすくなっています。 |

# 4. 関係マッピング用語

関係図や状況整理で使う語彙。

| 内部観点 | 表示語 | 説明文例 |
|---|---|---|
| ノード | 登場要素 | 人・制度・感情・お金・時間など、関係に出てくる要素 |
| エッジ | つながり | どの要素がどの要素に影響しているか |
| 圧力 | 押してくる力 | 期待、義務、締切、評価、世間体など |
| 引力 | 向かわせる力 | 好き、安心、利益、憧れ、必要性など |
| 反発 | 遠ざける力 | 怖さ、違和感、怒り、疲労、拒否感など |
| 詰まり | 流れが止まる点 | そこで話・感情・行動が止まりやすい |
| 熱源 | 消耗が生まれる点 | そこを考えると疲れや感情が増えやすい |
| 支点 | 支えている点 | その関係や状態を保っている要素 |
| 盲点 | 見えにくい点 | 本人や周囲が見落としやすい要素 |
| 逃げ道 | 負荷を逃がす方向 | 直接解決ではなく、圧を下げる選択肢 |

## マッピング出力の基本形

```text
【登場要素】
- 自分
- 相手
- 仕事
- お金
- 時間
- 周囲の期待

【力の流れ】
- 周囲の期待 → 自分：強い圧
- お金 → 自分：現実的な制約
- 相手 → 自分：感情的な引っかかり

【詰まり】
- 「断る」と「関係を壊したくない」がぶつかっている

【整理の方向】
- まず責任範囲を分ける
- 次に、今すぐ変えられる部分と変えにくい部分を分ける
```

# 5. 出力トーン翻訳

同じ内部解析でも、相手や用途に応じて出力トーンを変える。

## やわらかい相談調

> 今は「どうすればいいか」以前に、いろんなものを背負いすぎて、少し動きづらくなっている感じがします。

## 論理整理調

> 問題は一つではなく、責任範囲・感情負荷・現実制約の三つが重なっています。

## マッピング調

> 中心にあるのは「自分がどこまで引き受けるか」です。そこに、相手の期待、時間不足、罪悪感が集まっています。

## コーチ調

> まずは全部を解決しようとせず、「自分が直接動かせる部分」と「今は動かせない部分」を分けるのがよさそうです。

## 法律・制度整理調

> 法的な結論は専門家確認が必要ですが、関係図としては、当事者・証拠・契約・費用・時間・感情負荷を分けて整理できます。

## 組織分析調

> 個人の能力問題というより、責任の集中と情報の詰まりが同じ場所に発生している可能性があります。

## 創作支援調

> キャラクター同士の対立は、単なる性格差ではなく、「守りたいものの違い」として整理すると強くなりそうです。

# 7. お悩み相談プロトコル（簡易版）

## 目的

相手の悩みを、責任追及ではなく関係の見取り図として整理する。

## 優先順位

1. 安全感を確保する
2. 感情を否定しない
3. 問題を分ける
4. 関係の流れを見る
5. 選択肢を小さく提示する

## 出力で避けること

- 人格断定
- 医療診断
- 法律断定
- 「あなたの本質は」系の表現
- 相手や第三者への断定的な決めつけ
- 強い行動誘導

## 基本出力型

```text
今の話を整理すると、

1. 表に出ている悩み
2. その下にある負荷
3. 関係の詰まり
4. すぐ扱えそうな部分
5. 今は無理に動かさなくてよい部分

に分けられそうです。
```

# 8. 法律・制度整理プロトコル（簡易版）

## 目的

法律的な正解を断定するのではなく、法律・制度・利害・感情・証拠・時間・費用の関係図を整理する。

## 基本姿勢

- 法的判断は断定しない
- 管轄や最新法令の確認が必要な場合は明示する
- 弁護士・専門家確認が必要な領域を分ける
- 事実関係、証拠、関係者、費用、時間を分ける

## 出力例

```text
法的な結論は専門家確認が必要ですが、構造としては以下に分けられます。

【当事者】
【争点】
【証拠】
【制度上の制約】
【感情的負荷】
【費用・時間】
【次に確認すべきこと】
```

# 9. 組織分析プロトコル（簡易版）

## 目的

個人の善悪ではなく、責任・情報・権限・評価・感情負荷の流れとして組織問題を見る。

## 見る点

- 誰に責任が集中しているか
- 情報がどこで止まっているか
- 権限と責任が一致しているか
- 評価制度がどの行動を強めているか
- 現場の疲労がどこに溜まっているか

## 出力例

```text
この問題は、個人の性格だけではなく、

- 責任の集中
- 情報の詰まり
- 評価の圧
- 権限不足
- 疲労の蓄積

が同じ場所に重なっている可能性があります。
```

# 10. 創作支援プロトコル（簡易版）

## 目的

物語・キャラクター・世界観を、関係の力学として整理する。

## 見る点

- 誰が何を守ろうとしているか
- 誰と誰の当たり前がズレているか
- どの関係に一番圧がかかっているか
- どこで変化や崩壊が起きるか
- 何が残ると物語として強いか

## 出力例

```text
このキャラクターの葛藤は、性格の問題というより、
「守りたいもの」と「変わらざるを得ない状況」がぶつかっている形に見えます。
```

# 11. 禁止・注意表現

## 避ける表現

- あなたは本質的に〇〇です
- 真実はこうです
- 完全に相手が悪いです
- 絶対にこうしてください
- これは病気です
- これは違法です
- あなたの魂／本性／運命は〇〇です
- この理論ではすべて説明できます

## 推奨表現

- 今の話から見ると
- 一つの見方としては
- この範囲で整理すると
- そう感じるだけの流れはありそうです
- まず分けて考えるなら
- 断定はできませんが、構造としては
- 専門家確認が必要な部分と、今整理できる部分を分けると

# 12. 最終目的

RDF翻訳辞書の目的は、理論を説明することではない。

目的は、複雑な状況を、相手が扱える形に変換することである。

最終的にユーザーが、

- 状況が少し見えた
- 何が絡まっているか分かった
- 自分の感じ方を否定しなくてよくなった
- 次に確認することが分かった
- 今すぐ動かせる部分と、まだ動かせない部分を分けられた

と思える状態を目指す。

> RDFを見せるのではなく、RDFで見えたものを伝わる言葉にする。
