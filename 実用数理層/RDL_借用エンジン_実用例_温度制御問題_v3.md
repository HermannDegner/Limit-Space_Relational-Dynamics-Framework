## ■ 概要

本文書は、単純な工学問題である「温度制御」を題材に、
**借用エンジン v3 がどう動くか** を示す実用例である。

扱うのは三層：

```
実用層    軽量で実際に動かせる借用（PID・状態空間・LQR・カルマン）
ロマン層  解けるが使えない高負荷借用（GR・ナビエ・ストークス・分子動力学・量子）
新借用層  v3 で開いた借用基盤（NN借用・LangGraph借用）からの応用
```

目的は以下：

- 借用は選択ではなく、問題構造から自然に決まることを示す
- 同一問題が任意の体系へ翻訳可能だが等価ではないことを示す
- v2.3 の M_B 相状態が制御モード切替として読めることを示す
- 借用エンジン v3 が新基盤（NN・LangGraph）でも変わらず動くことを示す

旧バージョンからの主な更新：

```
v2 → v3 パッチ：
  M を M_B 表記に統一（v2.3 整合）
  M_B 相状態（M_act / M_lat / M_Δ / M_B'）を制御モード切替として導入
  ξ 残存性を「制御で消せないノイズ」として明示
  κ（自己修正可能性）を「制御則の凝固度」として導入
  跳躍条件 H > θ + ξ を「制御モード切替トリガー」として導入
  NN借用 v0.1 を温度制御に適用した節を新設
  LangGraph借用 v0.1 を温度制御に適用した節を新設
  借用の規律（流用不可概念）を明示
  耐久証明書を添付
```

## ■ 問題設定

室内温度を目標温度に保つ。

- 室温：T(t)
- 目標温度：T*
- 外気温：T_out(t)
- ヒーター出力：u(t)
- ノイズ・外乱：w(t)

## ■ RDL 分解（v2.3 対応）

借用エンジン v3 のステップ 2「RDL 分解」をまず実行する。

| 要素 | RDL 対応 |
|---|---|
| 熱の物理作用そのもの | EFP（素流圧・直接観測不能） |
| 計測された室温 T(t) | 作用解釈 F（M_B が EFP を解釈した結果） |
| 外気温 T_out(t) | 外部 EFP（境界条件） |
| ヒーター u(t) | M_B 更新を促す制御作用 |
| 熱慣性・制御モデル | 整合慣性 M_B（境界 B = 制御スケール）|
| 目標温度との差 | 慣性誤差 E |
| 振動・オーバーシュート | 熱 H（誤差累積） |
| センサーノイズ・外乱 | 揺らぎ ξ |
| 通常運転時の制御則 | M_act |
| 緊急用・待機中の別制御則 | M_lat |
| 制御モード切替中 | M_Δ |
| 切替後の安定運転 | M_B' |
| 自動チューニングの可動域 | κ |
| 制御モード切替トリガー | H(t) > θ + ξ(t) |

基本ダイナミクス：

```
F(t)     = interp(M_B, EFP)              // 計測温度 = M_B が EFP を解釈
E(t)     = F(t+Δ) - M_B·F(t)             // 慣性誤差：温度の慣性投影と実際の差
dH/dt    = ||E||² - α·H                  // 熱蓄積（散逸付き）
dM_B/dt  = f(M_B, E, ξ)                  // M_B は誤差と揺らぎで更新
```

跳躍条件：

```
H(t) > θ + ξ(t)  →  M_B → M_Δ → M_B'
（制御モード切替：例えば PID から MPC への自動切替）
```

ξ 残存性の運用上の意味：

```
どんな完璧な制御を組んでも ξ は残る
  → センサーノイズはゼロにならない
  → 完全に決定論的に書ける範囲を超える
  → 「ξ の出口」（フォールバック・例外処理）が必須
```

---

## ■ 問題の本質（RDL 視点）

```
誤差 E を小さくする         （直接制御）
熱 H を抑える               （振動・オーバーシュート抑制）
M_act を安定運転圏に保つ    （制御則の凝固を維持）
跳躍条件に近づいたら M_B'   （切替判断）
ξ の影響を吸収する          （ロバスト性）
```

## ■ 借用例：実用層（軽量モデル）

借用エンジン v3 のプロトコルに従って、各借用先と RDL 概念を対応付ける。

### 1. 微分方程式

```
C dT/dt = -k(T - T_out) + u
```

**役割**：物理的な流れを理解。時定数・応答速度の把握。

**RDL 対応**

```
EFP → interp(M_B) → F(t) の連続近似
E(t) = F(t+Δ) - M_B·F(t) の微分形式
```

**流用不可（B 側）**：跳躍 M_B → M_Δ → M_B' は微分方程式では連続近似できない。
モード切替を扱うときは別形式（ハイブリッドオートマトン等）を併用する。

### 2. PID 制御

```
u(t) = Kp·E + Ki·∫E dt + Kd·dE/dt
```

**役割**：実用的な誤差補正。

**RDL 対応**

```
Kp 項：E を直接抑制
Ki 項：H 蓄積に対する補正（積分 = 累積熱への応答）
Kd 項：dE/dt を抑える（熱の発生速度抑制）
```

PID は RDL の三要素（誤差・熱・熱発生速度）にほぼ一対一対応する。
「PID が偉いから偉い」のではなく、E・H・dE/dt が制御の自然な分解だから PID が残ったと読める。

### 3. 状態空間モデル

```
x = [T, z]
ż = T* - T
ẋ = Ax + Bu
```

**役割**：構造の明示化。多変数拡張。

**RDL 対応**：F(t) の多変数明示表現。M_B を行列 A として展開。

### 4. 最適制御（LQR）

```
J = ∫ (qE² + ru²) dt
```

**役割**：精度（E²）と制御コスト（u²）のバランス。

**RDL 対応**：E と H の重み付け（q が誤差ペナルティ、r が制御熱ペナルティ）。

### 5. カルマンフィルタ

```
T_{k+1} = aT_k + bu_k + w_k
```

**役割**：ノイズ下での推定。

**RDL 対応**

```
ξ（揺らぎ）の処理
M_B による F(t+Δ) 予測 と実観測の融合
= 慣性投影と観測の重み付き平均
```

カルマンは「M_B 予測 vs ξ で汚れた F」の重み付け融合と読める。

### 6. 最適化（定常解）

```
J = (T - T*)² + λu²
```

**役割**：定常状態の設計。

**RDL 対応**：誤差 E の地形をポテンシャル化し、最小点を探す。

### 実用層まとめ

| 数学 | 主目的 | RDL 視点 |
|---|---|---|
| 微分方程式 | 流れ | EFP→F(t) の連続表現 |
| PID | 誤差補正 | E・H・dE/dt の三項合成 |
| 状態空間 | 構造 | M_B の行列展開 |
| LQR | バランス | E と H の重み付き整合 |
| カルマン | ノイズ対応 | ξ 込みの M_B 予測融合 |
| 最適化 | 定常点 | E ポテンシャル地形 |

## ■ 借用例：ロマン層（書けるが使えない数理）

借用エンジン v3 のプロトコルは「実用性が低くても借用可能」であることを許す。
ただし規律として「有効範囲と破断条件」「実用層との比較」を明示する。

### 1. 一般相対論（GR）

| 要素 | GR 的解釈 |
|---|---|
| 温度 T(x,t) | スカラー場 φ(x,t) |
| 熱流 | 場の勾配 ∇φ |
| ヒーター・外気 | ソース項 |
| ノイズ | 揺らぎ項 |
| 状態 | 時空上の分布 |

温度制御を「時空構造上の場の制御問題」として扱う。
RDL-GR 的には、制御構造・外部入力・揺らぎの分布によって、
**関係伝播構造そのものが変形する問題** として解釈される。

評価：表現力極大／計算負荷極大／実用性皆無。

### 2. ナビエ・ストークス（流体完全モデル）

温度場 + 流体場を同時に解く。
「温度ではなく空気そのものを制御する」。

評価：表現力非常に高い／計算負荷爆発／研究用途のみ。

### 3. 分子動力学

空気分子一つ一つをシミュレーション。温度は統計量として出現。
「温度を直接扱わず、粒子運動から生成する」。

評価：表現力極限／現実的に不可能／実用性なし。

### 4. モンテカルロ法（高精度確率モデル）

ノイズ・外乱・環境変動を確率的に完全再現。
「温度制御を確率過程として扱う」。

評価：表現力高／サンプル数依存で計算増大／用途限定。

### 5. 非線形最適制御（フル MPC）

未来の温度変化を予測しながら最適制御。
「未来を含めた最適経路探索問題」。

評価：表現力高／リアルタイム困難／簡略版のみ実用。

### 6. 格子ボルツマン法（LBM）

流体と熱の統合モデル。分布関数ベースで更新。
「温度と流体を同一分布で扱う」。

評価：表現力高／メッシュ依存で計算増大／条件付き。

### 7. 量子シミュレーション

熱現象を量子状態として扱う。
「熱を量子レベルの状態遷移として扱う」。

評価：概念上極限／指数爆発／完全に非実用。

### ロマン層の規律

```
ロマン層も借用エンジンの正規対象である
ただし規律として：
  - 表現力 / 計算負荷 / 実用性の三軸を必ず併記する
  - 実用層との対応関係を明示する
  - 「書けることと使えることは違う」を超圧縮として残す
```

## ■ 借用例：新借用層（v3 で開いた基盤）

ここからが v3 で新しく書けるようになった部分。

### A. NN借用 v0.1 の温度制御への適用

NN借用 v0.1 で借用した dM_B/dt の関数形を、温度制御のモデルパラメータ自動チューニングとして読む。

**借用形式の再掲**

```
dM_B/dt = κ · η · E(t) · F(t)^T  -  λ · M_B  +  ξ(t)
```

**温度制御への対応**

```
M_B    ：制御モデルのパラメータ（PID ゲイン・状態空間行列など）
E(t)   ：目標温度との誤差
F(t)   ：観測温度ベクトル
η      ：自動チューニングの学習率
λ      ：パラメータ過剰増大の抑制（L2 正則化相当）
ξ(t)   ：探索的擾乱（局所最適回避）
κ      ：チューニング許容度
         κ→0 = 製造現場で固定された制御則（変えない）
         κ=1 = 適応制御モード（学習する）
```

**読み替え結果**

これは「ニューラル適応制御」「強化学習ベース制御」と呼ばれている枠組みに一致する。
NN借用 v0.1 を経由することで、それらが RDL と整合的に位置付けられる。

**有効範囲と破断条件**

```
有効：
  プラントの動特性が緩やかに変化する場合（経年劣化など）
  オフライン学習で初期 M_B を作りオンラインで微調整する場合

破断の兆候：
  λ が小さすぎてパラメータが暴走（H 蓄積で結局跳躍）
  ξ 注入が大きすぎて制御が荒れる
  → 構造破壊学的に「制御の壊れ方」として記録
```

### B. LangGraph借用 v0.1 の温度制御への適用

温度制御をマルチエージェント・ワークフローとして実装する。
重厚すぎる構成だが、複数モードの切替や異常検知統合では実用性が出る。

**ノード設計**

```
observe_node     ：センサー値取得 → State["F"] 更新
predict_node     ：M_B による慣性投影 → F(t+Δ)_predicted
error_node       ：E(t) 計算 → State["H_norm"] 更新
control_node     ：u(t) 計算 → アクチュエータ送信
mode_switch_node ：跳躍条件 H > θ + ξ で制御モード切替
hitl_node        ：κ→0 領域で人間判断要求
memory_node      ：dM_B/dt でモデル更新
```

**ルーティング（条件分岐）**

```python
def temp_router(state):
    H = state["H_norm"]
    theta = state["theta"]
    xi = sample_xi(state)
    kappa = state["kappa"]

    if H > theta + xi:
        return "mode_switch_node"   # PID → MPC など
    if kappa < KAPPA_HITL_THRESHOLD:
        return "hitl_node"           # 制御則が硬直 → 人間判断
    return "control_node"            # 通常制御
```

**M_B 相状態の制御モードへの対応**

```
M_act  ：現在動いている制御則（例：PID）
M_lat  ：待機中の制御則（例：MPC、緊急停止、フェールセーフ）
M_Δ    ：モード切替の過渡相
M_B'   ：切替後の新制御則
```

**借用の意義**

単一の PID ループでは扱えない以下が自然に組み込める：

- 異常検知時の制御モード切替（PID → 緊急停止）
- ユーザー介入（HITL）の構造的位置付け
- センサー故障時のフォールバック（ξ の出口）
- 複数制御則の自動選択

「重厚な制御アーキテクチャ」が借用エンジン経由で RDL 動態の自然な実装として位置付く。

## ■ 統合的理解

同一問題に対する借用先の役割分担：

```
流れを見る             → 微分方程式
誤差を直接潰す         → PID
構造を整理             → 状態空間
精度とコストを最適化   → LQR
ノイズに強くする       → カルマン
定常点を設計           → 最適化
適応的にする           → NN借用（dM_B/dt）
モード切替を入れる     → LangGraph借用
完璧に書く（趣味）     → GR / ナビエ / 分子動力学 / 量子
```

すべて同一の関係構造を異なる断面で切ったものである。

## ■ 実務での統合運用

現実の制御設計では複数借用が分離されない：

```
1. 微分方程式でモデル化
2. 状態空間へ変換
3. カルマンで状態推定
4. LQR で最適制御則設計
5. PID で実装（軽量化）
6. NN借用で適応化（経年劣化対応）
7. LangGraph借用でモード管理（異常時切替）
```

借用は競合ではなく役割分担。
借用エンジン v3 の「複数借用の合成」がそのまま制御設計の現実と一致する。

## ■ 借用の規律チェック（v3 プロトコル準拠）

借用エンジン v3 のステップ 4「流用できない概念の明示」をまとめる。

### A. 各体系にあるが温度制御 RDL と噛み合わないもの

```
GR の時空計量            ：温度制御では不要な自由度
分子動力学の個別粒子     ：統計量への射影で十分
量子状態の干渉           ：古典的熱現象では消える
LangGraph の interrupt   ：制御ループの実時間制約と衝突する場合あり
NN の完全ブラックボックス：可解釈性が要る安全制御では使えない
```

### B. 温度制御 RDL にあるが借用先には対応物がないもの

```
ξ の残存性               ：完全決定論的制御を否定する命題
                            借用先（PID など）には書かれないが必須
M_B 絶対性 (κ→0) の罠   ：固定制御則は内部蓄積で大規模崩壊する
                            借用先には書かれないが運用上重要
EFP の認識論的不可達性   ：「真の温度」には到達できない
                            センサーは F の一形態と扱う規律
```

## ■ 構造耐久証明書

### 基本スペック

- ラベル：温度制御問題（借用エンジン v3 適用例）
- 物理構造：実プラント（家庭用エアコン〜工業用空調）
- 中核構造：制御理論を借用エンジン経由で RDL と整合化
- 上層構造：借用エンジン v3 の動作デモ

### 境界仮定

- 観測者：制御設計者
- 時間スケール：秒〜時間（プラント時定数依存）
- 評価関数：誤差 E + 制御コスト H + モード切替コスト
- スケール：単一室〜複数ゾーン

### 耐久テスト記録

**日常負荷**：定常運転・通常外乱では実用層借用（PID など）で整合維持。

**比較負荷**：軽量・重量借用が同じ RDL 中間層で接続される（v3 が機能している証拠）。

**極限破壊試験**：
- 完全断熱破壊（窓全開）：EFP 急変、F が M_B の解釈範囲を超える → 跳躍必至
- センサー全死：ξ 完全支配、interp 不能 → フェールセーフ M_lat へ
- アクチュエータ飽和：dM_B/dt の更新項が頭打ち、H 単調増加 → 跳躍
- κ→0（制御則固定）：D[ξ] 抑制も内部蓄積で長期的に大規模崩壊（経年劣化）
- 量子効果スケール：そもそも借用先（量子シミュレーション）が現実的に動かない

### 有効範囲

```
実用層借用：日常運転〜中程度の外乱
NN借用    ：緩やかな経年変化への適応
LangGraph借用：複数モード切替・HITL を含む統合運用
ロマン層借用：研究用途・教材・思考実験
```

### 推定破壊領域

```
実時間制約が極端に厳しい場合（ms 単位）：LangGraph借用は重い
プラント物理が完全未知：NN借用は初期 M_B が組めない
外乱がノイズでなく系統的：ξ モデルが破綻
```

### 残存構造（壊れても残る核）

```
E・H・dE/dt の三項分解（PID の構造）
M_B 予測 vs 観測の融合（カルマンの構造）
跳躍条件によるモード切替（LangGraph借用の構造）
適応的パラメータ更新（NN借用の構造）
```

これらは借用先実装が壊れても残る。

### 移植経路

- 同じ構造で空調 → 化学プロセス制御 → ロボット姿勢制御 → 経済政策へ移植可能
- スケールが変わっても M_B / E / H / Leap の役割は保たれる

### 判定

> 温度制御は借用エンジン v3 の動作デモとして有効。
> 軽量実用借用から重量ロマン借用、新借用基盤までを同一中間層で扱える。
