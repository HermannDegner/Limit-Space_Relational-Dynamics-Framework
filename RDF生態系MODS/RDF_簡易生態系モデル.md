RDF的 生態系NPC AI 構築設計書（詳細設計版・整理改訂）
1. 概要

本ドキュメントは、関係力学言語（RDF）に基づき、生態系NPCエージェントが「環境」という素流圧の海の中で、いかに**安定（整合）を維持し、いかに変化（跳躍）**へ応答するかを定義するものである。

本モデルにおいて、NPCは固定的な行動表を持つ存在ではない。
NPCはそれぞれ異なる構造を持ち、環境から到来する素流圧を受信し、内部で熱を生成・解消しながら、局所的な整合を維持するように振る舞う。これはRDFの基本である「構造・入力・出力・更新」の循環モデル、および整合と跳躍の二動態に対応する。

2. 設計思想
2.1 世界観の前提

本モデルでは、世界は「物の集合」ではなく、作用の流れが構造に影響を与える場として扱う。
NPCが受け取るのは客観的な真実そのものではなく、常に局所的・制約付きの素流圧である。この考え方は、観測が作用を介してしか成立しないという限界空間的前提と整合する。

2.2 基本目的

各エージェントの基本目的は、単純な意味での「勝利」ではない。
それぞれの構造にとっての熱の管理と整合維持こそが基本目的である。

熱が高すぎれば跳躍が起きる
熱が低すぎれば停滞し、退屈や探索圧が発生する
資源や安全は熱の解消手段である
行動とは、熱と整合の動的バランスの結果である
3. 環境構造（Environment Structure）

世界は単なる座標系ではなく、素流圧を媒介・変形・遮蔽する物理構造として設計される。

3.1 地形

地形は移動可能性だけでなく、観測可能性や情報伝播効率を規定する。

岩・崖・水場
移動ベクトルの制限
視線遮断
匂い・音の伝播変形
平地
高い移動効率
低い遮蔽性
長距離視認を許す
3.2 資源

資源は単なるHP回復要素ではなく、内部熱の解消源として扱う。

草（資源）
短い草
採餌効率が高い
遮蔽率が低い
被食者にとって露出熱が上がりやすい
長い草
採餌効率が低い
遮蔽率が高い
捕食者の隠密行動を支援する
被食者にとっては安心と不安の両義性を持つ
3.3 風（情報のベクトル場）

風は空間全体に方向性を与えるベクトル場であり、匂いや音の伝播効率を変える。

風上：情報の源泉が見つかりやすい
風下：情報が流れ去り、観測の死角となる

風の導入により、環境は単なる静的マップではなく、時々刻々と情報の価値が変わる場になる。

4. 観測システム（Sensory Dynamics）

エージェントは複数種の素流圧を受信するが、それぞれ時間軸・精度・意味が異なる。
この差が行動多様性を生む。

4.1 匂い
RDF的定義：過去の整合慣性の残響
特徴：
時間減衰する
空間に場として残る
正確性は低いが履歴性が高い
役割：
追跡
過去の存在推定
「そこにいた」の感知

匂いは「今ここ」の確定情報ではなく、過去の構造が残した痕跡である。

4.2 音
RDF的定義：瞬発的な素流圧パルス
特徴：
突発的
不連続
強い熱上昇を誘発しやすい
役割：
危険通知
群れ同期
行動中断

音は、精密な対象認識というより、誤差を一気に増幅する起爆刺激として機能する。

4.3 視覚
RDF的定義：局所的な高解像度確定情報
特徴：
狭角
高精度
遮蔽に弱い
役割：
ターゲット固定
行動予測補正
誤差最小化

視覚は「確定性」が高いため、恐怖源にも安心源にもなりうる。
見えている捕食者は危険であると同時に、「位置が読める」という意味で熱を抑えることもある。

5. エージェント構造（Agent Structure）

各エージェントは、構造行列、基層パラメータ、熱、閾値、関係強度を持つ。

5.1 基本変数

各個体 $i$ は少なくとも以下を持つ：

$A_i$ ：構造（入力を変換するルール）
$H_i$ ：現在熱
$\Theta_i$ ：跳躍閾値
$p_i$ ：基層パラメータ群
$r_{ij}$ ：対象 $j$ への関係強度
$\hat{x}_{t+1}$ ：予測状態

基本力学はRDF数理モデルに従い、入力と誤差から熱が更新され、熱が閾値を超えたとき構造更新が起きる。

6. 個体設計
6.1 ウサギ（Rabbit）― 脆弱な高感度構造
基層構造
ノルアドレナリン $n_t$ ：極大
セロトニン $s_t$ ：極小

この設計は、危険兆候への感度が高く、誤差許容が低い構造に相当する。基層パラメータの近似的意味づけとしても整合的である。

挙動特性
小さな音・匂いでも熱が急上昇
跳躍閾値が低く、逃走が優先される
視認できない危険に対して仮想熱を持ちやすい
安全が続きすぎると退屈熱が発生し、探索へ移る
行動優先順位
近距離危険回避
群れ同期逃走
採餌
漫然待機ではなく、退屈熱に応じた移動
6.2 キツネ（Fox）― 攻撃的な探索構造
基層構造
ドーパミン $d_t$ ：高
セロトニン $s_t$ ：中〜高
関係強度 $r_t$ ：対象固定時に上昇

これは誤差を探索し、失敗を許容しつつ、対象に重みを乗せて追跡を継続できる構造である。

挙動特性
匂いから痕跡追跡
風下・遮蔽を利用した接近
視認前は「存在を確定させない」ことを優先
一度ロックした対象に対して感度が上がる
成功捕食が最大の熱解消となる
行動優先順位
痕跡の取得
隠密接近
対象ロック
攻撃実行
失敗時の再探索
6.3 ハヤブサ（Falcon）― メタAI・強制整合装置
役割
世界全体の熱 $H_{world}$ の監視
過密、停滞、資源枯渇の兆候に応じて介入
生態系全体の排熱担当
特性
回避困難な速度
個体単位では対処困難な物理圧
捕食というより、システム安定のための強制削減
意義

ハヤブサは単なる強敵ではなく、世界側が持つメタ的な整合回復機構である。

7. 相互作用の力学（Interaction Dynamics）
7.1 観測不確実性と熱
露出した捕食者
視覚的に位置と状態が把握できる
危険ではあるが予測可能性が高い
被食者側で局所線形近似が成立しやすい
熱は上がるが暴走しにくい
隠れた捕食者
観測不能
予測が分散
最悪事態の仮想誤差が膨らむ
熱が持続的に蓄積する

この違いは、単なる可視・不可視ではなく、整合可能か否かの違いである。SILNの成立／崩壊という形で整理できる。

7.2 警告の伝播（構造同期）
一頭の個体が高熱状態に入る
警告音という強パルスを放つ
周囲の個体がそれを危険素流圧として受信
群れ全体の熱が同期上昇
一括して逃走構造へ移る

これはRDF的には、群れの相転移である。

7.3 負の素流圧（隠密）

キツネは「自分の情報量を下げる」ことで優位に立つ。
これは単に見つからないことではなく、相手にとっての誤差計算材料を減らすことであり、予測不能性を高める操作である。

8. 状態遷移モデル（実装向け）

実装上は各個体に有限状態を持たせつつ、その遷移条件を熱と感覚入力で決めるとよい。

8.1 ウサギの状態例
Rest
Graze
Investigate
Alert
Escape
遷移例
Rest -> Graze : 退屈熱上昇
Graze -> Alert : 匂い/音の閾値超過
Alert -> Escape : 熱 $H_i > \Theta_i$
Escape -> Rest : 一定時間危険観測なし
8.2 キツネの状態例
Wander
Track
Stalk
Chase
Feed
遷移例
Wander -> Track : 匂い検知
Track -> Stalk : 対象位置の推定収束
Stalk -> Chase : 視認確定
Chase -> Feed : 成功捕食
Chase -> Wander : ロスト

10. 実装上の重要ポイント
10.1 「知能らしさ」は複雑な推論からではなく熱管理から出す

複雑な意思決定木よりも、

何で熱が上がるか
何で熱が下がるか
何を見ているか

を先に設計した方が、生物らしさが出る。

10.2 環境の方を賢くする

AI個体だけを賢くするのではなく、

風
遮蔽
痕跡
資源再生
を入れることで、個体の行動が自然に豊かになる。
10.3 状態よりも遷移条件が本体

RDF的には状態名そのものより、
どんな素流圧と熱条件で状態が崩れるか
の方が重要。

11. 応用可能性

本モデルは動物生態系に限らず、そのまま社会モデルへ写像可能である。

ウサギ型：危機管理、ゼロトラスト、防災
キツネ型：競争戦略、マーケティング、欺瞞構造
ハヤブサ型：法規制、外部圧力、破壊的変化

これはRDFが「存在」ではなく「作用と整合」を中心に置くためである。

12. 総括

本設計の本質は、NPCを「行動表で動く駒」としてではなく、

素流圧の中で整合を維持しようとする構造

として記述する点にある。

生態系とは、複数の構造が相互に熱を与え合い、局所的安定と非連続変化を繰り返す動的場である。
したがって、本モデルが生成するものは単なるNPC挙動ではなく、現象そのものである。
