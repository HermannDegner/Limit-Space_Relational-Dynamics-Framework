# RDF 汎用エージェントエンジン 実装全体計画書（改訂版 MVP）

## 1. 目的

本計画書は、RDF（Relational Dynamics Framework）に基づく汎用エージェントエンジンを、
**段階的に実装・検証するための実行計画**を定義する。

改訂版では、レビュー指摘を反映し次を強化する。

1. Actionを独立終端ではなく「外部化器（Projection + Actuator）」として扱う
2. Jumpを単発フラグではなく状態機械（primed/testing/fixing）で扱う
3. `M` は将来の行列化に備え、InertiaModel抽象を先に置く
4. Neuroは「熱生成層」ではなく「勾配変形層」と明記する
5. 学習更新は後段ではなくPhase 1から微小版を導入する

---

## 2. 設計原則

- **構造更新の基底**: `S / H / M / ξ / Jump` を単一更新系として扱う
- **層責務の固定**:
  - Sensor: 未解釈作用束の正規化
  - Neuro: 勾配の再重み付け・歪み付与
  - Base: 状態積分（`S/H/M/ξ`）
  - Projection: 外部化する行動候補の選定
  - ActionPlugin: 行動の世界適用（実行器）
- **拡張性**: MVPは簡略実装、型は将来拡張可能に維持

---

## 3. 目標アーキテクチャ（改訂）

```text
Sensor -> Neuro -> Base -> Projection -> ActionPlugin(Actuator)
```

### 3.1 層責務（実装上の定義）

- `Sensor Layer`
  - 世界状態を受け取り、Base可読形式の勾配束へ変換
- `Neuro Layer`
  - 入力勾配束を、基層パラメータ・期待・誤差履歴・退屈熱で再重み付け
  - **有効勾配（effective gradient）** を生成
- `Base Layer`
  - `S/H/M/ξ` を更新
  - Jump状態機械を進行
  - 微小学習（慣性更新）を適用
- `Projection Layer`
  - 現在状態から外部化候補（移動・発話・内部探索など）を選択
- `ActionPlugin`
  - Projection結果を実際の環境操作へ適用

## 4. データモデル（最小）

```python
@dataclass
class AgentState:
    S: np.ndarray
    H: float
    M: "InertiaModel"
    mode: str                 # "base" | "nam" | "jump"
    jump_state: str           # "none" | "primed" | "testing" | "fixing"
    jump_candidates: list[Any]
    jump_history: list[Any]
    boredom: float = 0.0
    last_projection: str | None = None
```

```python
class InertiaModel(Protocol):
    def apply(self, s: np.ndarray) -> np.ndarray: ...
    def update(self, delta: Any) -> "InertiaModel": ...
```

- MVP実装: `ScalarInertiaModel`
- 将来拡張: `DiagonalInertiaModel` / `MatrixInertiaModel`

```python
@dataclass
class GradientBundle:

```

## 5. 不変条件（invariants）

`docs/invariants.md` に最低限以下を固定する。

1. `H >= 0`
2. `M` は負の暴走を起こさない（実装依存で上限/正則化を持つ）
3. Jump完了後は `H` が減少またはリセットされる
4. Sensor出力はBase可読形式へ正規化される
5. Layer依存は上流 -> 下流の単方向
