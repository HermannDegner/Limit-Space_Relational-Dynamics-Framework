# LS文書規約

*プロジェクト全体に適用される文書作成・管理のルール*

---

## ■ 1. ファイル名

**バージョン番号をファイル名に含めない。**

```
OK ：RDL_Core.md
NG ：RDL_Core_v1.1.md
```

バージョン情報は文書内部（ヘッダーとフッター）に記載する。
ファイル名が安定することで、他文書からの参照が更新不要になる。

---

## ■ 2. 文書ヘッダー

文書の冒頭に以下の形式で記載する。

```
# 文書タイトル
*Tier / カテゴリ / ステータス vX.Y*
*依存：[T1以上の依存文書を列挙]*
```

例：

```
# RDL_Core
*T1：Core層 / CORE / ACTIVE v1.1*
*依存：（T0は暗黙前提のため省略）*
```

```
# RDL_創造論
*T2：運用層 / OP / ACTIVE v0.3*
*依存：RDL_Core / SILN / 構造破壊学_前提文書*
```

上位文書がある場合は追加する：

```
*上位：RDL_中間設計図プロトコル*
```

---

## ■ 3. T0の扱い（暗黙前提）

**T0（哲学的基底層）は全文書の暗黙前提である。**

T0文書（限界空間・形而上空間・文書性の限界・仮設という操作）は、
LS体系全体の基底として機能するため、依存ヘッダーへの個別記載は不要。

```
OK ：*依存：RDL_Core / SILN*
OK ：（T0前提のみの場合）*依存：（T0暗黙前提）*
NG ：*依存：限界空間 / 形而上空間 / RDL_Core / SILN*
          ↑ T0を毎回列挙しない
```

T0の特定概念を強調したい場合は本文中で参照すればよい。

## ■ 5. バージョン記載

バージョンはヘッダーとフッターに記載する。

**ヘッダー**：`*T2：運用層 / OP / ACTIVE v0.3*`

**フッター（版記録）**：

```
*v0.3：[この版で変更した内容の要約]*
*v0.2：[前版の変更内容]*
*v0.1：初版。[初版の内容]*
```

バージョンの意味：

```
v0.x ：草案・実験段階
v1.x ：安定版
v2.x ：大幅改訂版
x.0  ：構造的変更を含む更新
x.y  ：内容追加・修正
```

## ■ 6. 文書地図への登録

新規文書を作成したら `文書地図.md` に追記する。

記載内容：文書名・カテゴリ・ステータス・一行概要。

---

## ■ 7. バージョン更新時の手順

文書を大きく更新する場合、旧版をアーカイブに残す。

```
1. 現行ファイルを旧バージョン名でアーカイブにコピー
     例：RDL_Core.md → 99_Archive/01_T1/RDL_Core_v1.1.md

2. 現行ファイルを更新する（ファイル名はそのまま）
     例：RDL_Core.md（v1.2として更新）

3. フッターに版記録を追記
```

アーカイブ内のファイルはバージョン番号をファイル名に残す。
「いつの版か」が名前だけで判別できることが重要なため。

## ■ 8. フォルダ構造

```
00_T0_哲学的基底層/
01_T1_Core層/
02_T2_運用層/
03_T3_外部接続層/
04_T4_応用層/
05_T5_メタ層/
06_TD_共有語彙対応層/
90_Tool_and_Code/
99_Archive/
  00_T0/
  01_T1/
  02_T2/
  03_T3/
  04_T4/
  05_T5/
  06_TD/
  その他/
```

アーカイブはアクティブ層と同じTier構成。
「この文書の旧版はどこ」が直感的に分かる。

アーカイブへの移動：現役でなくなった文書は対応するTierフォルダへ。
ステータスを `LEGACY` に変更してから移動する。
アーカイブ内ではファイル名にバージョン番号を残す。
