# 仮設という操作
*T0：哲学的基底層 / CORE / ACTIVE*
*依存：限界空間 / 形而上空間 / 文書性の限界*

---

## ■ 0. なぜこの文書が必要か

RDL体系の至る所に「仮設」という語が現れる。

```
SILNを仮設する
Bを仮設する
M_Bを仮設する
EFPを仮設すると便利
形而上接続操作としての仮設置
```

しかし「なぜ仮設なのか」「仮設とは何をする操作か」を正面から扱う文書がなかった。

本文書はその問いに答える。

---

## ■ 1. なぜ「発見」ではなく「仮設」か

限界空間の前提：

> 観測者が直接持てるのは作用解釈Fだけであり、対象そのものには到達できない。

これが「仮設」の根拠である。

対象に直接到達できるなら、正しい記述を「発見」できる。しかし限界空間の内側では、観測者は常にFから逆算するしかない。逆算の結果は「正しい答え」ではなく「整合的な仮説」である。

```
発見  ── 対象に到達して取り出す操作。限界空間内では不可能
仮設  ── Fから逆算して構成する操作。限界空間内で唯一可能な操作
```

だから「仮設」は謙虚さや暫定性の表明ではない。

**限界空間の構造から導かれる、唯一の操作様式である。**

---

## ■ 2. 仮設の構造

仮設は以下の要素を持つ。

```
1. 目的B         ── 何のために仮設するか
2. 構成          ── Fから整合的な構造を逆算する
3. 運用          ── 仮設した構造で動かす
4. 誤差の観測    ── 仮設と現実のズレを見る
5. 更新または破棄 ── 誤差に応じて仮設を引き直す
```

この5ステップはSILN分解の循環操作と同一の構造である。

仮設は一回の操作ではなく、誤差・更新・再仮設のサイクルとして機能する。

---

## ■ 3. 仮設と確定の違い

| | 仮設 | 確定 |
|---|---|---|
| 前提 | 限界空間（到達不能） | 絶対基底（到達可能） |
| 操作 | Fから逆算して構成 | 対象から直接取り出す |
| 耐久性 | 誤差に応じて更新される | 変わらない |
| 引き直し | 可能・必要 | 不要 |
| 形而上空間との関係 | 限界空間内で機能する | 形而上空間に属する |

確定は形而上空間への越境である。限界空間の内側では、すべての記述は仮設として扱われる。

---

## ■ 4. 仮設の射程

体系内で「仮設」が機能している場所：

**T0：哲学的基底**
- 限界空間そのものが「絶対的確定性を仮設しない」という仮設
- 形而上空間は「仮設置」という接続操作で扱う

**T1：Core**
- M_Bは「仮設された整合慣性」
- EFPは「仮設すると便利な操作的概念」

**T2：運用層**
- SILN分解はBとM_Bを仮設する循環操作
- 創造論はBを仮設→破壊→再仮設のサイクル

**T3：外部接続層**
- 外部理論の借用は「この範囲でRDLと整合する」という仮設的接続

**T4：応用層**
- SFOは性格を「現在の観測条件における仮設的断面」として扱う
- 奏和市の素流圧配置は「このBで市民はこう動く」という仮設に基づく

**T5：メタ層**
- 幸福モデルのM_B柔軟性テストは「M_Bを絶対化せず仮設として保持できるか」を問う
- 文書性の限界は「すべての文書は近似＝暫定的な仮設」と主張する

---

## ■ 5. 仮設であることの強さ

「仮設」は弱さではない。

仮設であるから：

```
・引き直せる
・誤差を受け取れる
・壊れても再構成できる
・閉じない
・生きている
```

確定した記述は誤差を受け取れない。誤差が来ても「現実の方が間違っている」として処理される。これがM_Bの過剰固定化であり、拘束型不幸の構造であり、構造破壊学が扱う破断条件の一形態である。

仮設として保持することが、構造の耐久性を高める。

---

## ■ 6. 仮設と形而上空間の境界

仮設はどこまでも仮設である。

「これは仮設ではなく確定だ」と感じるとき、それは形而上空間への越境のサインである。

形而上接続操作（指し示す・仮設置・信じる）は、仮設を動機として機能させるために使う。しかし限界空間内に「確定」として持ち込むと体系が崩れる。

```
「このM_Bは正しい」  ── 形而上越境
「このM_Bは今耐久している」  ── 仮設として保持
```

この区別が運用上の最重要ポイントの一つである。

---

## ■ 7. 一文圧縮

> 仮設とは、限界空間の内側で唯一可能な操作様式であり、到達不能な対象に対してFから整合的な構造を逆算し、誤差に応じて引き直し続ける動的操作である。

---

## ■ 8. 超圧縮

> 確定できないから仮設する。仮設だから引き直せる。引き直せるから閉じない。閉じないから生きている。
