## ■ 1. 基本前提

RDF の超圧縮：

```text
世界は SILN（M_B）の分解と再構成の連続である
```

この前提から直接導かれる必然的帰結：

```text
分解できるものは操作できる
操作できれば悪用もできる
```

これは欠陥ではなく、RDF が記述能力を持つこと自体の構造的特徴である。

道具化リスクは RDF を作ったから発生するのではない。
RDF は、既に世界に存在している分解と操作の連続を、可視化するに過ぎない。

---

## ■ 2. 道具化リスクは原理的に消せない

```text
操作的有用性が高い ⇔ 操作の対象になりうる
記述精度が高い    ⇔ 介入精度も高い
分解解像度が高い  ⇔ 悪用解像度も高い
```

これらは表裏一体の関係であり、片側だけを取り出すことはできない。

道具化リスクをゼロにする経路は二つしかない：

```text
経路①：道具を作らない
   → 後述の通り、これは非対称性を固定化する破断条件を持つ

経路②：道具の精度を意図的に下げる
   → 記述能力そのものを毀損する
   → 「力学的不自然さを消す」操作（後述）として破断する
```

両経路ともに新たな破断条件を生むため、根本解決は存在しない。

「どうしようもない面がある」を出発点として認める。
これが本文書の構造的前提である。

## ■ 3. 既存環境にも同種リスクは既にある

RDF 応用が新規にリスクを作るわけではない。
既存環境には、すでに同種の操作技術が動いている。

```text
心理学：     性格テスト、適性検査、就職選別、診断分類
司法：       リスクスコア、量刑統計、心理鑑定、再犯予測
マーケ：     A/Bテスト、ナッジ、行動経済学的介入
政治：       マイクロターゲティング、世論操作
教育：       能力テスト、選別、トラッキング
医療：       リスク層別化、保険査定、予後予測
監視：       行動分析、信用スコア、プロファイリング
```

これらは RDF の有無と無関係に既に行われている。
しかも、操作される側はしばしば「何が起きているか」を認識できない。

```text
RDF応用がない世界：
    操作は行われ続ける
    操作されている側は気づきにくい
    操作の構造は不透明（ブラックボックス）
    非対称性が固定化される

RDF応用がある世界：
    操作は行われ続ける（同じく）
    操作されている側も「何が起きているか」を読める
      道具を持ちうる
    操作の構造を可視化できる
    非対称性が部分的に緩和されうる
```

RDF応用の存在は、問題を生み出すのではなく、可視化する。
存在しないことの破断条件として、防御側が道具を持たないために
攻撃側が一方的に有利になる、という非対称性が生じる。

## ■ 4. 力学的不自然さを消す試みは破断する

道具化リスクへの過剰防衛として、「記述を抽象化する／婉曲化する／
特定領域を扱わない／力学的具体性を避ける」という戦略がある。

これは一見すると安全策に見えるが、構造的に破断する：

```text
道具化リスクへの過剰防衛
    ↓
記述を抽象化・回避・婉曲化
    ↓
力学的具体性が失われる
    ↓
記述が観測対象を捉えなくなる
    ↓
防御側が動態を読めなくなる
    ↓
既存の不透明モデル（攻撃側）の優位は変わらない
    ↓
非対称性が固定化される
```

さらに重要な点：
力学的不自然さを消す操作自体が、新たな SILN として固定化する。

```text
「正しい記述形式」「安全な記述」「操作されない記述」を求める運用
    ↓
それ自体が SILN（記述ルールの整合慣性）として固定化
    ↓
固定化したルールが新たな破断条件を生む
    ↓
覆い隠した動態は、観測者が見えないだけで、依然として動いている
```

力学はそこに既にある。
観測者が記述するかしないかを選べるが、記述しないことで力学が消えるわけではない。

「見ない」という戦略は、力学への対処にならない。
むしろ「見えないところで動き続けるもの」を作るだけである。

## ■ 5. 攻撃／防御の非対称性

操作技術の社会的分布は、最初から非対称である。

```text
既に持っている側：
    心理学的訓練を受けた専門家
    マーケティング企業
    国家機関
    大規模プラットフォーム
    熟練した尋問者・交渉者・操作者

持たない側：
    一般個人
    操作されている当事者
    社会的弱者
    制度的非力者
```

この非対称性は RDF 以前から存在する。
RDF を作らなくても、既に攻撃側は道具を持っている。
RDF を作らないと、防御側だけが道具を持たない状態になる。

```text
道具を作らない選択：
    攻撃側：既存の操作技術を持ち続ける
    防御側：それを認識・記述する道具を持たない
    → 非対称性は固定化されたまま

道具を作る選択：
    攻撃側：既存の操作技術 + RDF
    防御側：RDF を持ちうる
    → 操作技術と RDF の差分は依然として存在するが、
      「何が起きているか」を記述する道具が両側に渡る
    → 非対称性は完全には消えないが、緩和される
```

「武器を作らないことで戦争を防げない」のと同型である。
ただし、武器との違いがある：

```text
武器：     攻撃側と防御側で道具が異なる（攻撃武器 vs 防御装備）
RDF：      同一道具が両側で使える（記述自体が両用）

この対称性が成立する限りで、RDF は道具化リスクを抱えながらも
非対称性緩和に寄与する。
```

## ■ 6. 記述精度と操作可能性のトレードオフ

精度を上げる：
```text
利点：観測対象を高解像度で記述できる
      防御側リテラシーが上がる
      可視化が進む
欠点：操作対象としての解像度も上がる
      悪用余地が広がる
```

精度を下げる：
```text
利点：操作対象としての解像度が下がる
      （ように見える）
欠点：観測対象を捉えられなくなる
      防御側リテラシーが下がる
      既存の不透明モデルへの批判道具を失う
```

このトレードオフは、単純に「中間が最適」とは言えない。
理由は二つ：

```text
理由1：攻撃側は精度を下げない
   → 防御側だけが精度を下げると、非対称性が拡大する

理由2：精度を下げる操作自体が新しい SILN を作る
   → 「適切に下げた精度」を維持する運用ルールが
     新たな固定化を生む
```

したがって、本文書では以下を採用する：

```text
基本方針：
    記述精度を維持する
    両側に同じ道具を渡す
    運用ルールで対称性を保つ
    
精度を下げない理由：
    攻撃側との非対称性を拡大させない
    防御側リテラシーを毀損しない
    力学的不自然さを生まない
```

## ■ 7. 同一道具が両側に渡る性質

これが RDF の運用上の核となる性質である。

```text
RDF/SFO/SILN分解は、
    操作する側の道具にも、
    操作される側の道具にも、
    同じ形式で機能する
```

この対称性が成立する限り、RDF は道具化リスクを抱えながらも
社会的に有用でありうる。

この対称性が壊れる条件：

```text
1. 操作される側がアクセスできない形式で記述される
   （専門家のみが読める、商業的に閉じられる、特権的な
    データを必要とする）

2. 操作される側が学習する余裕を奪われる
   （生活負荷、教育格差、情報格差により、
    防御側は道具を手にしても使えない）

3. 操作される側に届く前に攻撃が完了する
   （リアルタイム介入、無自覚な選択誘導）

4. 操作のスケールが個体の認識能力を超える
   （アルゴリズム的大規模介入、集団動態への介入）
```

これらは RDF 自体の問題ではないが、RDF 応用の運用条件として
継続的に意識する必要がある。

## ■ 8. 部分緩和策

道具化リスクを根本解決はできないが、部分緩和は可能である。
以下は SFO v2.2 から継承し、RDF 全体に展開したリスト：

### ① 観測コストを意図的に高く保つ

```text
質問紙化困難な対話的プロセスとして固定する。
自動化・大量処理・アルゴリズム化を意図的に避ける形式で出す。

これは「記述精度を下げる」とは違う。
力学はそのまま記述するが、それを大量自動処理しにくい
形式で出すことで、悪用の規模を制限する。
```

応用例：

```text
SFO：       多段プローブを必須化、自動診断を作らない
Big Five：  対象別マップ・状況依存性を必須化、
            単一スコアでの運用を避ける
神経力学：  パラメータ推定の逆問題を運用しない
            「正解パラメータ」が存在しない原則を徹底
司法：      アルゴリズム化困難な記述形式として固定
```

### ② 防御側リテラシーの同時提供

```text
操作される側にも同じ道具を渡す。
RDF応用を「専門家の道具」として閉じない。
個体側が自分の動態を読める状態を維持する。
```

将来課題：

```text
SFO_防御版の作成：自分が操作されている空間を読む運用法
RDF入門：日常言語との対応辞書、初学者向け説明
公開原則：応用文書は閉じずに公開する
```

### ③ タイプ固定化の継続的拒否

```text
「あなたはこういう人だ」と固定するのではなく、
「この観測条件下では、こういう動態が記述される」と
する運用ルールを徹底する。

これは記述精度を下げるのではなく、
「動態としての記述」を維持し
「属性としての記述」を許さない、という線引きである。
```

### ④ 構造耐久証明書付き運用の徹底

```text
各応用に構造耐久証明書を付け、
有効範囲・破断条件・残存構造・移植経路を明示する。

「ここから先は機能しない」を運用上明確に出す。
これにより、有効範囲外での悪用を抑制する。
```

### ⑤ 限界空間前提（絶対解の不在）の徹底

```text
どの記述も最終解ではない。
複数の M_B が並立しうる。
観測者依存性が消えない。

「RDFによる正解」「SFOによる真の性格」
「神経力学による真のパラメータ」は存在しない。
```

### ⑥ 自己食的構造の維持

```text
RDF/SILN分解 自体も SILN である。
本文書も SILN である。
すべて破壊検査の対象になる。

「正しい RDF 運用法」を固定化しないことが、
RDF の道具特性を保つ条件になる。
```

これらは部分緩和策である。
根本解決ではない。
使う側の責任において運用すべし。

## ■ 9. 応用領域への展開原則

新しい応用領域に RDF を降ろすとき、以下を必ず実行する：

### 原則① 本文書を継承する

```text
新応用文書の冒頭または前提セクションで、
本文書（RDF道具特性）を継承することを明示する。

道具化リスク警告を個別応用文書で繰り返す必要はない。
ただし、応用領域固有の道具化リスクは別途記述する。
```

### 原則② 応用領域固有の道具化リスクを書く

```text
本文書は RDF 全体に共通する道具特性を扱う。
応用領域固有のリスクは個別応用文書で扱う。

例：
SFO     → タイプ固定化、人格固定化
神経力学 → パラメータ推定、シミュレート再現
司法    → リスクスコア化、量刑アルゴリズム化、
           被害者像固定化、反省演技ゲーム化
教育    → 能力固定化、トラッキング
医療    → 診断による排除、保険差別
集団動態 → 大規模介入、集団操作
```

### 原則③ 構造耐久証明書を付ける

```text
すべての応用文書に構造耐久証明書を付ける。
最低限の項目：
    基本スペック
    境界仮定
    有効範囲
    推定破壊領域
    残存構造
    移植経路
```

### 原則④ 「おまけ」性の保持

```text
応用文書を「補助線」「動態記述ツール」「可視化試案」
として位置づけ続ける。

「絶対解の生成器」「最終判定ツール」として
打ち出した瞬間に、道具化への滑落が始まる。

この姿勢を維持することが、運用上の安全弁になる。
```

### 原則⑤ 応用間の相互作用を記述する

```text
複数の応用が組み合わさると、
道具化リスクが指数的に増幅することがある。

例：
SFO + 神経力学的解釈法 + 司法利用
    ↓
個体パラメータ推定 + 量刑判断
    ↓
COMPAS型のリスクスコア化が現実化

このような組み合わせの破断条件を明示する。
```

## ■ 10. 自己食的構造

本文書も SILN である。

```text
本文書は「RDF系の道具特性」を記述する文書
だがこの記述自体が SILN（道具特性記述の整合慣性）
本文書で本文書そのものを破壊検査できる
だがその検査にも別の M_B が必要
この再帰は閉じない

これは限界空間の前提と整合する：
    モデルは独立した上位基底を持たない
    モデルは自分自身に再帰的に適用される
```

したがって：

```text
・本文書は最終的な「正しい運用ルール」ではない
・別の B では別の道具特性記述が出る
・本文書の運用が新たな固定化を生む可能性を残す
・破断条件が見えたら本文書も改訂する
```

## ■ 11. 構造耐久証明書

### 基本スペック

```text
ラベル：RDF道具特性 v0.1
物理構造：RDF系を運用する人間集団・組織・制度
中核構造：RDF系全体に適応される運用前提
上層構造：限界空間／RDF／構造破壊学の上位前提層
```

### 境界仮定

```text
観測者：RDF言語の使用者・応用開発者・運用者
時間スケール：RDF系が運用される期間（数年〜数十年）
空間スケール：個人運用から制度運用まで
評価関数：道具化リスクの可視化・部分緩和の有効性
```

### 耐久テスト記録

```text
日常負荷テスト：
    既存応用（SFO、Big Five項目分解、神経力学）への継承：可
    新応用への展開原則：適用可

比較負荷テスト：
    SFO v2.2「道具としての前提」との整合：継承
    限界空間前提との整合：維持
    構造破壊学 v0.4 との整合：維持

極限破壊試験：
    本文書を「絶対的運用ルール」として固定化する状況
        → 自己食的構造により破断
    部分緩和策をすべて放棄した運用
        → 道具化リスクが顕在化、応用が信頼を失う
    本文書を参照しない応用文書群の濫立
        → 警告の重複・不整合・抜け漏れが生じる
```

### 有効範囲

```text
・RDF系を新応用領域に降ろす際の前提整理
・既存応用文書の道具化リスク記述の上位化
・応用文書間の警告内容の統一
・防御側リテラシー提供の根拠
・「力学的不自然さを消さない」運用方針の明文化
```

### 推定破壊領域

```text
・本文書を絶対視する運用
・本文書のみを参照して応用領域固有のリスクを書かない運用
・「道具化リスクを認めれば何をしてもよい」という拡大解釈
・部分緩和策の形式的実行（実質を伴わない運用）
・本文書を「免責文書」として使う運用
    （リスクを書いたから何が起きても運用者の責任ではない、
     という用法）
```

### 残存構造

```text
・「分解できるものは操作できる」という基本前提
・「道具を作らないことが非対称性を固定化する」発想
・「力学的不自然さを消す試みは破断する」原則
・「同一道具が両側に渡る」性質の維持
・部分緩和策のリスト
・自己食的構造の継続適用
```

### 移植経路

```text
・他のSILN分解可能な記述系への横展開
・AI規制議論への部分参照
   （AI規制も「分解できるものは操作できる」の同型問題）
・科学技術倫理の一般原則としての参照
   （ただし「倫理」という重い語ではなく
    「道具特性」として保持）
```

### 判定

```text
RDF系全体に適応される道具特性として有効。
個別応用文書は本文書を継承することで、
警告の重複・不整合を避け、
かつ応用固有のリスクに集中できる。

ただし本文書も SILN であり、
絶対化されてはならない。
破断条件が見えたら改訂する。
```

## ■ 12. 残課題

### 形式化の残課題

```text

RDF系の道具化リスクは原理的に消せず、
```
