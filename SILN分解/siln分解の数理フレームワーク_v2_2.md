# SILN分解の数理フレームワーク v2.2
*（RDF概念的数理モデル v2.3 対応／S除去・EFP-F分離・時間差誤差・慣性投影パッチ適用済み／ξ+B+M_B 基底化・ラベル統一パッチ適用済み／ラベル M_B ネットワーク動性・追従運用化パッチ適用済み）*

---

## ■ 概要

SILN（Survival-biased Integration of Local Linear approximations in a Nonlinear world）は
そのままでは抽象度が高すぎるため、分解なしでは意味を持たない。

したがって、実用上は

**「境界 B を仮定し、整合慣性 M_B を仮設する操作」**

として扱う。

※ M_B は「抽出」ではなく「仮設」である。直接観測できないため、
　行動・出力・反応のパターンから推定した仮説に過ぎない。

※ v2.1 で明示：M はすべて境界 B 依存の M_B である。
　独立した「M そのもの」は存在しない。
　SILN 分解は本質的に「B の選択 + M_B の仮設」のセット操作である。

※ v2.2 で明示：SILN分解は一回操作ではなく **追従運用** である。
　ラベル・B・M_B のすべてが時間発展するため、
　分解はスナップショットではなく循環的な引き直しになる。

## ■ 全体プロセス（v2.2 改訂）

```
① ラベル設定（対象定義）
② 境界 B の仮定（仮説）
③ 状態展開（M_B, EFP, F, E, H, ξ）
④ 構造仮設（M_B の推定）
⑤ 動性追跡（ラベル・B・M_B の揺れの観察）
⑥ 引き直し（①〜④の循環）
```

※ v1.1までのS(t)はM_Bの読み出し断面に過ぎないため除去。
　必要な場合はM_Bの断面として取り出す。

※ v2.1 で明示：① と ② は分離可能だが独立ではない。
　ラベルの選択は B の暗黙的選択を含み、B の選択は観測者自身の M_B に依存する。

※ v2.2 で明示：⑤⑥ は v2.1 までは暗黙だった循環構造を明示化したもの。
　SILN分解は「分解完了」という終端を持たない。

## ■ ① ラベル設定（対象定義）

対象となるSILNをラベルとして定義する。

例：
- 知性
- 国家
- 貨幣
- テセウスの船

※ この段階ではまだ中身は存在しない
※ ラベル自体が SILN（ラベル運用の整合慣性）であることに注意

**v2.2 補足：ラベルは固定点ではない**

```
ラベルは独立した記号ではない
ラベル = 多数の M_B が結節するノード
ラベルの「中身」 = 接続している M_B たちのネットワーク

このネットワークは常時揺れている
したがって「ラベル設定」も静的な確定操作ではない

詳細は本文書「ラベルの M_B ネットワーク動性」節を参照
```

## ■ ② 境界 B の仮定（仮説）

SILN分解における最重要ステップ。

以下を仮定する：

- 空間スケール（個体／社会／物理）
- 時間スケール（瞬間／人生／文明）
- 評価関数（生存／効率／快）
- 用途・目的

```
B = 観測者が引いた境界の組
M_B = P_B(整合慣性場)
```

※ 限界空間では絶対前提は存在しないため必須

**v2.1 補足：B と観測者の関係**

```
B は観測者が選ぶ
だが「観測者」自身も M_B である
したがって B の選び方は観測者の M_B から派生する

観測者 = M_{B_observer}
B の選択 = M_{B_observer} の動態の出力

純粋に外側から B を引く観測者は存在しない
これは限界空間の構造的非対称：「境界を引く」行為だけは irreducible
```

**v2.2 補足：B の動性**

```
B は引かれた瞬間から揺れ始める
B は静止しない（詳細は概念的数理モデル v2.3「B の動性」を参照予定）

ξ残存性と B の動性は一対：
  ξ残存性     ：B を引いても ξ は M_B の中・縁・外に残る（B の不徹底性）
  B の動性    ：引いた後も B は静止しない（B の非静止性）

B は二重に閉じない
したがって SILN分解も二重に閉じない（→ 追従運用化）
```

## ■ ③ 状態展開

### ● 整合慣性（SILN本体）
```
M_B
```

- 習慣
- 制度
- 構造パターン
- 認識のクセ

M_B は独立して観測できない。
行動・出力・反応のパターンからの仮設に過ぎない。

**M_B の相状態**

```
M_act  : 活性断面（現在強く機能している）
M_lat  : 潜在断面（まだ発現していない）
M_Δ    : 再編相（変化の途中）
M_B'   : 再編後

これらは別の M ではなく、同じ M_B の異なる相状態
```

---

### ● 素流圧（EFP：Elementary Flux Pressure）
```
EFP
```

未解釈の作用束。M_B の外側から流入する。

**重要：EFPは直接観測できない**
```
観測者が実際に持つのはF（作用解釈F）だけである
EFPはFからM_Bを使って逆算した推定概念に過ぎない

EFP_estimated = interp⁻¹(M_B, F)

𝒦と同じ位置づけ：到達不能だが操作上有用なため仮設置する
```

---

### ● 作用解釈F
```
F = interp(M_B, EFP)
```

M_B が EFP を解釈した結果。
観測者が実際に持つ唯一の作用情報。

```
知覚の質はM_Bに依存する
M_BがリッチなほどEFPから多くを読み取れる
M_BがフラットならEFPはほぼ素通り
```

---

### ● 誤差（慣性誤差）
```
E(t) = F(t+Δ) - M_B·F(t)
```

- `M_B·F(t)`：M_B の慣性が F(t) に乗ったまま次へ流れた値（**慣性投影**）
- `F(t+Δ)`：実際に来た次のF
- `E(t)`　 ：慣性投影と実際のFの差 = 慣性誤差

**Δ（時間差）の意味：**
```
Δ = 0 は完全確定 = 限界空間で扱えない
Δ > 0 が必然

Δ小　：因果が明確・M_B 更新しやすい（鉄球衝突）
Δ大　：因果が曖昧・M_B 更新しにくい（キノコ毒・エキノコックス）
Δ∞　：因果不明・M_B は更新されない（温暖化・ゆでガエル）
```

---

### ● 熱
```
H = ||H_vec||
H_vec ≡ δM_B(t)　// M_B の未拘束自由度における振動
```

慣性誤差の蓄積。
H は E の時系列積分として蓄積する。

---

### ● 揺らぎ
```
ξ(t)
```

確率的変動・根源的自由度。
D[ξ] = 𝒦 / ||M_B|| により慣性と交換可能。

**v2.1 補足：ξ の残存性**

```
B を引いても ξ は M_B の中・縁・外に残る
D[ξ] = 𝒦 / ||M_B|| ≠ 0 が常に成立

つまり SILN分解で M_B を仮設しても
その M_B には常に ξ が染み込んでいる
分解結果が「動的」であるのはこのため

詳細は概念的数理モデル v2.3「ξの残存性」節を参照
```

---

## ■ 基本ダイナミクス

```
F(t)    = interp(M_B, EFP)         // EFP を M_B が解釈
E(t)    = F(t+Δ) - M_B·F(t)        // 慣性誤差
dM_B/dt = f(M_B, E, ξ)              // M_B は慣性誤差と揺らぎで更新される
```

**流れ：**
```
EFP（仮設・直接観測不能）
    ↓ interp(M_B, EFP)
F(t)（作用解釈F・観測者が持つ唯一の情報）
    ↓ M_B·F(t) 慣性投影
    ↓ F(t+Δ) が実際に来る
E(t) = F(t+Δ) - M_B·F(t)（慣性誤差）
    ↓
H蓄積 → 跳躍（M_B → M_Δ → M_B'）
```

## ■ ④ 構造仮設（推定）

SILN分解の目的は M_B の構造を仮設すること。
M_B は直接観測できない。あくまで観測パターンからの推定である。

### ● 固有分解

```
M_B = V_B Λ_B V_B⁻¹
```

- V_B：安定方向（境界 B における慣性の向き）
- Λ_B：強度（各方向の慣性の強さ）

### ● 相状態の取り出し

```
M_act  = active(M_B^ij)
M_lat  = latent(M_B^ij)
M_Δ    = M_B が再編相にあるときの動的断面
```

## ■ ⑤ 動性追跡（v2.2 新設）

SILN分解結果は静的構造ではない。
分解後も以下のすべてが揺れ続ける：

```
ラベル          ：ラベル M_B ネットワークの再配線
B               ：境界の動性（v2.3 概念モデル参照）
M_B             ：相状態遷移（M_act / M_lat / M_Δ / M_B'）
M_B 内の ξ      ：残存・蓄積・解放
観測者の M_B    ：観測者自身が変化することで分解結果も変動
```

### ● 追跡対象

- ラベルの中身（接続 M_B ネットワーク）の変化
- B が局所固定として持つ有効性の劣化
- M_B の相状態遷移
- ξ 蓄積の進行（H 増加）
- 跳躍条件への接近（H > θ + ξ）

### ● 追跡が機能するための条件

```
追跡そのものが新たな M_B（追跡運用の SILN）として固定化しないこと
固定化した瞬間に追跡装置自体が観測対象から外れる
追跡装置にも κ（自己修正可能性）が必要
```

これは RDF 道具特性文書の「自己食的構造」と整合する。

## ■ ⑥ 引き直し（v2.2 新設）

動性追跡が一定の閾値を超えると、SILN分解全体を引き直す。

### ● 引き直しトリガー

```
(1) ラベル M_B ネットワークの実質的変化
    例：「水」というラベルの化学的中身が変わった
(2) B の有効性劣化
    例：適用スケールから対象が逸脱した
(3) M_B の相転移
    例：M_B → M_Δ → M_B' が完了した
(4) 観測者の M_B 変動
    例：観測者の学習・経験により B の引き方が変わった
```

### ● 引き直しのフロー

```
動性追跡 → 閾値超過検知 → ①〜④ を新条件で再実行
```

### ● 重要：完全再構築ではない

```
引き直しは部分更新で良い
ラベルだけ更新するケース
B だけ更新するケース
M_B だけ更新するケース
全部更新するケース
```

どの層で再仮設が必要かは追跡結果から決まる。

## ■ ラベルの M_B ネットワーク動性（v2.2 新設・最重要）

> **ラベルは記号ではない。ラベル = M_B が結節するノードである。**

これは v2.2 で明示化される最重要追加命題である。

### ● ラベルの実体

ラベルは独立した記号ではない。
ラベル = 多数の M_B が結節するノード。
ラベルの「中身」 = そのノードに接続している M_B たちのネットワーク。

**例：ラベル「水」**

```
M_{B_化学}    : H2O・分子構造
M_{B_物理}    : 流体力学的対象
M_{B_生体}    : 生命維持物質・血液成分
M_{B_文化}    : 清浄・浄化の象徴
M_{B_経済}    : 商品・資源
M_{B_日常}    : 飲むもの・洗うもの
M_{B_慣用}    : 「水商売」「水に流す」
...
```

これらの接続パターン全体が「水」というラベルの中身を構成する。
ラベルそれ自体は中身を持たない。
**接続パターンそのものが意味になる。**

### ● ネットワークが揺れる理由

```
(1) 個別 M_B の揺れ
    各 M_B が B の動性により揺れる
    → ノードに接続している全 M_B が同時に揺れている

(2) 接続関係 C_ij の揺れ
    M_{B_化学} ↔ M_{B_生体} の接続強度
    M_{B_文化} ↔ M_{B_日常} の接続強度
    これらも時代・文脈で変動する

(3) ラベル運用 SILN の揺れ
    「水」という記号をどう使うかのルール = ラベル運用 SILN
    これも M_B なので揺れる
```

### ● 動態の典型パターン

**① ノード追加**
```
新概念の出現 → 新 M_B が結節
例：「クラウド」「サブスク」「推し」
```

**② ノード枯死**
```
M_B が衰退 → 「死語」「古語」化
ラベル自体は残るが接続が枯れる
例：「ナウい」「チョベリバ」
```

**③ 接続再配線**
```
ラベルは同じだが接続先 M_B が変化
例：「やばい」
  旧接続：M_{B_危険}, M_{B_不都合}
  新接続：M_{B_肯定的強調}, M_{B_感嘆}
  → 接続反転が起きている
```

**④ ネットワーク分岐**
```
一ラベルが複数の独立した M_B ネットワークに分裂
例：「リアル」（実在 ↔ 対面 vs オンライン）
```

**⑤ ネットワーク融合**
```
別ラベルの M_B 群が同一ラベル下に統合
例：「サステナブル」が「エコ」「環境配慮」「持続可能」を吸収
```

### ● 国語辞書という観測装置

国語辞書は、ラベル M_B ネットワークの **時間断面スナップショット** である。

```
辞書 = 時刻 t における観測記録
辞書改訂 = ネットワーク変化の事後追認
```

**重要：辞書更新は欠陥ではなく機能要件**

```
不変辞書 → ラベル M_B ネットワークの追従に失敗
       → 観測装置として機能停止
       → 規範化（あるべき意味の押し付け）に転落
```

辞書は「正解の記録」ではなく「現時点での近似」。
SILN 分解の応用文書と全く同じ位置づけ。

### ● 帰結

```
・「同じラベル」の中身は時刻によって異なる
・ラベルの連続性は同一性を保証しない
・テセウスの船問題はこの命題から自動的に解ける
・AI のバージョン更新も同じ構造で扱える
・辞書・百科事典・分類体系は全て時間断面スナップショット
```

### ● 規範化リスク

ラベルに静的な中身を持たせようとする運用は規範化に転落する：

```
辞書 → 正解の記録として運用       → 規範化
タイプ分類 → 真の人格として運用    → 固定化
ラベル → 実体の名前として運用      → 静止化
```

ラベル M_B ネットワークの動性を認める運用が、
RDF 系を RDF 系のまま保つ条件。
詳細は RDF 道具特性文書を参照。

## ■ 動性の階層（v2.2 整理）

LSRDF における動性は以下の階層構造を持つ：

```
[1] M_B の動性
    M_act / M_lat / M_Δ / M_B' の相状態遷移
    （v2.1 で既出）

[2] B の動性
    境界そのものが静止しない
    ξ残存性と対の基底命題
    （概念的数理モデル v2.3／v2.4 で扱う）

[3] ラベル M_B ネットワークの動性
    ラベルに接続する M_B 群の再配線
    （本文書 v2.2 で新設）

[4] 観測者の M_B の動性
    観測者自身も M_B なので揺れる
    上記すべての動性を観測者依存にする
```

すべて「絶対基底が無い」の異なるスケールでの現れ。
**動性は欠陥ではなく構造的帰結である。**

## ■ パラドックスの解体（v2.2 拡張）

```
argmax Identity_B が複数存在
```

にも関わらず単一解を要求するため破綻する。

本来、事象は複数の M_B（SILN）として並立可能である。
単一の絶対解へ収束させようとすることで整合が破綻する。

**v2.1 補足：パラドックスとは B の問題**

```
多くのパラドックスは B を固定せずに「絶対解」を求めることから生じる
B を明示すれば、複数の答えが「異なる B における異なる M_B」として並立できる
パラドックスは限界空間の仕様に対する前提（B の暗黙的固定）の不適合を示す現象
```

**v2.2 補足：パラドックスの再記述**

```
パラドックスは三層から生じる：

(a) B を固定せずに絶対解を求める（v2.1 既出）
(b) B の動性を無視して B を静止扱いする
(c) ラベル M_B ネットワークの動性を無視してラベルを実体扱いする

テセウスの船 = (c) の典型
ソリテス・パラドックス = (b) の典型（「砂山」の B が連続変動）
自己同一性パラドックス = (a)〜(c) の重畳
```

## ■ 重要性質（v2.2 拡張）

- SILN は境界依存（M_B として記述される）
- M_B の定義＝仮説
- M_B は観測者依存
- EFP は直接観測不能・F からの逆算推定
- 誤差は慣性投影と実際のFの時間差
- Δの大きさが因果の認識可能範囲を決める
- 同一性は連続量
- 分解しないと意味を持たない
- ξ は M_B の中・縁・外に残り続ける（v2.1：ξ残存性）
- B を引く行為自体が観測者の M_B から派生する（v2.1：B の内発性）
- SILN分解結果は静的構造ではなく M_B → M_Δ → M_B' という動的相状態を含む（v2.1）
- **ラベルは記号ではなく M_B ネットワークの結節点である（v2.2）**
- **ラベル M_B ネットワークは常時揺れ続ける（v2.2）**
- **SILN分解は一回操作ではなく追従運用である（v2.2）**
- **動性は欠陥ではなく LSRDF の構造的帰結である（v2.2）**

## ■ SILN分解の限界性

**M_B の検証問題**

ラベルの M_B を強固にするには、複数の観測者から収集する操作が有効である。
しかしその操作自体が EFP として M_B に作用する。
したがってラベルの M_B は観測プロセスと不可分であり、
観測前の純粋な M_B には到達できない。

**絶対解は存在しない**

```
M_{B_観測者A} ≠ M_{B_観測者B} ≠ M_{B_観測者C} ...
```

複数観測によってラベルの輪郭は統計的に近似できるが、
真の M_B への収束は保証されない。

SILN分解は絶対解を出すツールではなく、
**目的に応じた有効な近似を仮設するツール**である。

限界空間において確定は局所的確定に過ぎない。
SILN分解もその例外ではない。

**v2.1 補足：自己食的構造**

```
SILN分解は M_B を仮設する操作
だがこの「分解」自体も SILN（分解運用の整合慣性）
SILN分解で SILN分解そのものを分解できる
だがその分解にも別の M_B が必要
この再帰は ξ の残存と同様に閉じない

これは限界空間の前提と整合する：
モデルは独立した上位基底を持たない
モデルは自分自身に再帰的に適用される
```

**v2.2 補足：追従運用の限界**

```
SILN分解を追従運用化しても閉じない問題は残る：

(1) 追跡装置自体が SILN 化する
(2) 追跡頻度・解像度の選択にも別の M_B が要る
(3) 「いつ引き直すか」の判定基準も M_B
(4) 完全な動性追従は原理的に不可能

したがって追従運用も近似であって完全解ではない
「動性を扱える」ことと「動性を完全に追跡できる」ことは別物
```

## ■ 同一性の再定義

```
Identity_B = similarity(M_B(t), M_B(0))
```

→ 同一性は物理ではなく M_B の持続で決まる
→ 同一性自体も B 依存である（B が違えば「同じか違うか」の判定も変わる）

**v2.2 補足：同一性とラベル M_B ネットワーク**

```
ラベル M_B ネットワークは変化している
→ ラベル連続性は同一性を保証しない
→ 同一性判定にはラベル抜きで M_B ネットワーク自体を見る必要がある
```

## ■ テセウスの船への適用（v2.2 拡張）

```
SILN = (M_B, L, W)
```

- M_B　：整合慣性（境界 B 依存の SILN本体）
- L　　：ラベル
- W　　：社会的関係

※ v1.1のS_phys は M_B の物理断面として読み直す

本質：

```
船 = M_B(t) の軌跡
B = 「船」というラベルが運用される境界
```

観測者ごとに B が微妙に異なるため M_B も微妙に異なる。
造船士の B、乗組員の B、港の人の B、それぞれ別の M_B。
それでも「テセウスの船」というラベルが共有できるのは、
M_B が完全一致しなくても機能する共有装置としてラベルが働くからである。

**v2.2 補足：ラベル M_B ネットワーク動性からの自動解**

```
「テセウスの船」ラベルに接続する M_B ネットワーク：

  M_{B_物理} : 部品構成
  M_{B_歴史} : 歴史的連続性
  M_{B_機能} : 船として機能するか
  M_{B_文化} : 神話的同一性
  M_{B_所有} : 法的所有者

部品交換は M_{B_物理} を変化させるが
M_{B_歴史}・M_{B_機能}・M_{B_文化}・M_{B_所有} はほぼ不変

「同じ船か」の答えは
どの M_B に接続を重く置くか（B の選択）で決まる
→ 絶対解は構造的に存在しない
```

## ■ ラベルの二面性

```
① 境界 B を作り、思考の自由度を下げる
② 観測者間の M_B のズレを束ね、共有を可能にする
```

M_B が完全に一致していればラベルは不要。
完全にバラバラならば共有できない。
**微妙にズレているちょうどその状態**でラベルが意味を持つ。

ラベルとは観測者間の慣性誤差を吸収するバッファである。

同じ対象から、B が違えば違う M_B が現れる。
