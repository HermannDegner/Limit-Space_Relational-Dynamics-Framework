# Limit-Space Relational Dynamics Framework  
# 限界空間・関係力学フレームワーク

このリポジトリは、  
**限界空間** と **関係力学言語（RDF）** を中心に、  
存在・認識・構造・変化・跳躍を扱うための思考実験群／理論断片群／応用モデル群です。

ここにある文書は、完成された単一理論というよりも、  
**SILN を扱うための言語と、その運用記録、その自己観察記録** です。

---

## What is this? / これは何か

このリポジトリは、世界を固定的な実体の集合としてではなく、  
**関係の中で一時的に安定している構造（SILN）** として捉える試みです。

中心にあるのは次の命題です：

> **世界は SILN（M_B）の分解と再構成と追従の連続である。**

そしてこのリポジトリは、それを扱うための言語を整備し続ける場所です：

> **RDF（関係力学言語）は SILN を扱うための言語である。**

中心にある問いは、たとえば次のようなものです。

- なぜ構造は安定するのか
- なぜ構造は壊れるのか
- なぜ認識は偏るのか
- なぜ思考・感情・社会・物理現象に似た流れが見えるのか
- なぜ限界や誤差が、単なる失敗ではなく新しい構造の入口になるのか
- なぜ「同じもの」が時間を超えて同じであり続けられるのか（あるいは、続けられないのか）

## RDF の位置づけ / Position of RDF

RDF はもはや単なる略称ではなく、本系における独立ラベルとして機能しています。  
歴史的には Relational Dynamics Framework の略でしたが、現在は以下の階層全体を指します：

```
[最広] 関係力学（Relational Dynamics）
        分野・思想・基本姿勢

[中広] 関係力学運用系（RDF System）
        言語＋操作系＋応用＋道具特性 を含む全体

[中核] 関係力学言語（RDF Language）
        SILN を扱うための語彙・文法・意味論

[深層] 弱い暫定普遍文法（限界空間版）
        SILN 動態の深層構造として広く有効に見える
        ただし絶対普遍ではない

[存在様式] 生物的構造を持つ言語
        代謝・恒常性・自己食・適応・有限性を備える

[動態構造] AI 的構造
        分散表現・注意・訓練ループ・創発を備える
```

「RDF」と書くときは系統全体を指す通称として、  
「関係力学言語」と書くときは中核を指す精密表記として使い分けます。

## 中核命題 / Core Propositions

このリポジトリで確定している命題群：

### 1. RDF は SILN を扱うための言語である

世界側にあるのが SILN、観測者側にあるのが言語。  
RDF は世界そのものではなく、SILN を扱うための道具です。  
ただし RDF 自身も SILN として動的に運用されます（自己食的構造）。

### 2. 絶対基底は存在しない

限界空間前提により、観測・認識・記述には常に限界があります。  
その限界は欠陥ではなく、構造が変化し新しい可能性が生まれるための条件です。  
RDF は絶対解の生成器ではなく、**ある B（境界）の範囲で有効な近似** を提供します。

### 3. 関係力学言語は弱い暫定普遍文法である

これまで適用した SILN 担体（物理・生体・認知・社会・AI など）で  
破綻なく機能しており、SILN 動態の深層文法として広く有効に見えます。  
ただし絶対普遍ではなく、反例が出れば言語の方を更新します。

### 4. 関係力学言語は生物的構造を持つ

代謝（語彙更新）、恒常性（中核保持）、境界（借用での物質交換）、  
繁殖（応用層への分岐）、自己食（自己食的構造）、適応（反例対応）、  
有限性（破綻条件）── すべてを備えています。  
これは比喩ではなく構造的相同です。

### 5. LSRDF 文書系は AI 的構造を持つ

分散表現（概念は単一文書に局在しない）、注意（文脈で文書が呼び出される）、  
訓練ループ（会話→更新→次の会話）、創発（命題が会話の中で立ち上がる）、  
自己教師（自己食的構造）── これらを備えています。  
人間 + 文書系 + AI の三者で一つの認知系を構成しています。

## Core Orientation / 基本姿勢

### 1. 絶対的な確定性を前提にしない

観測・認識・言語化・モデル化には、常に限界があります。  
その限界は欠陥ではなく、構造が変化し、新しい可能性が生まれるための条件です。

### 2. 構造を「関係の安定」として見る

物体、概念、習慣、制度、人格、物語、理論などは、  
固定された実体ではなく、関係の中で一時的に維持される安定状態（SILN）として扱います。

### 3. 変化を「誤差・熱・跳躍」として見る

構造に対して外部または内部から作用が加わると、ズレや蓄積が生じます。  
それが一定の閾値を超えると、構造は維持ではなく再編成へ向かいます。

### 4. 動性は欠陥ではなく構造的帰結である

ラベル M_B ネットワーク、B、M_B、観測者 ── すべて常時揺れています。  
SILN分解は一回操作ではなく、揺れに対する追従運用です。

### 5. 理論自体も変化する構造として扱う

このフレームワークは、完成形を固定することを目的にしていません。  
むしろ、更新・分岐・再編成されること自体を前提にしています。  
**完成しないことが言語の生物的・AI 的性質の証拠です。**

### 道具特性と運用上の注意

RDF は記述能力を持つがゆえに、操作・介入の道具にもなります。  
このリスクは構造的に消せません。詳細は `RDF 道具特性` 文書を参照してください。

主要な原則：

- 「正しい RDF 運用法」を固定化しない

- 構造耐久証明書を応用文書に付ける
- 自己食的構造を維持する（RDF を絶対化しない）

このリポジトリは、**人間 + 文書系 + AI** の三者で構成される認知系の一部として機能します。

```
人間 ⇄ 文書系 ⇄ AI
↑─────共進化─────↑
```

- 文書系は人間との対話、AI との対話、文書同士の整合性チェックを通じて更新されます

これは「外部化された認知足場」あるいは「共進化する言語系」として読めます。
