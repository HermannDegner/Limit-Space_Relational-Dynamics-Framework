# RDF 人間用アタッチモデル仕様（軽量版）

## 概要
本仕様は、関係力学言語（RDF）のベース数理モデルに対して、人間固有の反応特性・感情出力・意識的微修正を扱うための「アタッチ層」を追加するための軽量数理仕様である。

本仕様の立場は以下である。

- ベース数理は普遍的な動態を扱う
- 人間モデルはその上に接続される解釈・重み付け・更新制御モジュールである
- 人間モデルはベース数理を置き換えず、誤差の意味づけと更新方向を補助する
- 数式形式は固定せず、運用目的に応じて選択・併用できる

この仕様は、RDF数理モデルの整合・誤差・跳躍の枠組みを保ったまま、人間の感情・学習・関係性を記述するための最小構成を示す。

---

## 1. 設計思想

### 1.1 コアとアタッチの分離
ベース数理は以下のような普遍的動態を扱う。

- 構造
- 入力
- 誤差
- 整合
- 跳躍
- 更新

人間用アタッチ層は以下を追加する。

- 誤差の重み付け
- 関係性による熱量変化
- 基層構造による反応特性
- 意識による観測と微修正
- 感情という出力形式

### 1.2 基本立場
本仕様では以下を前提とする。

- 素流圧からの入力は、ほぼ確実に誤差を含む
- 誤差は熱として感じられる
- 整合とは熱の解消ではなく、熱を構造に変換して低エネルギー化する過程である
- 整合された熱は整合慣性として固定される
- 人間の意識は主処理ではなく、誤差観測と微修正を担う

### 1.3 数理形式の選択可能性
本仕様で提示する式は、唯一の正解として固定するためのものではなく、操作的な最小記述として置かれている。

したがって実運用では、対象領域・データ粒度・目的関数に応じて、以下を自由に選択または併用してよい。

- 線形近似モデル
- 非線形動力学モデル
- 確率モデル
- 情報理論的指標
- 離散時間更新則

重要なのは式の形そのものではなく、誤差の熱化、重み付け、微修正、整合慣性、跳躍という対応関係が保たれていることである。

---

## 2. ベース数理

人間用アタッチは、以下のベースモデルに接続される。

```math
x_{t+1} = A_t x_t
```

- `x_t` : 入力された素流圧
- `A_t` : 時刻 `t` における構造
- `x_{t+1}` : 構造を通した出力

誤差は予測との差として定義する。

```math
e_t = x_{t+1} - \hat{x}_{t+1}
```

- `e_t` : 誤差ベクトル
- `\hat{x}_{t+1}` : 構造が事前に想定していた出力

---

## 3. 熱の定義

人間モデルでは、誤差はそのまま熱になるのではなく、重み付けされた誤差として熱化する。

```math
H_t = \| W_t e_t \|
```

- `H_t` : 時刻 `t` の熱量
- `W_t` : 人間アタッチ層による重み付け行列

ここでの意味は以下である。

- 同じ誤差でも、人によって熱量は違う
- 同じ人でも、相手や状況によって熱量は違う
- 関係性や基層構造が、何を重く感じるかを決める

---

## 4. 人間用アタッチパラメータ

### 4.1 基層パラメータベクトル
人間用アタッチ層の最小パラメータとして、以下を置く。

```math
p_t =
\begin{bmatrix}
d_t \\
s_t \\
o_t \\
n_t
\end{bmatrix}
```

- `d_t` : ドーパミン的探索・報酬傾向
- `s_t` : セロトニン的誤差許容度
- `o_t` : オキシトシン的自己境界拡張度
- `n_t` : ノルアドレナリン的危機増幅度

### 4.2 役割
各パラメータの機能は以下の通り。

#### ドーパミン的パラメータ `d_t`
- 誤差探索を促進する
- 誤差解決時の快を強める
- 学習率や探索性を上げる

#### セロトニン的パラメータ `s_t`
- 誤差に対する許容度を上げる
- 同じ誤差でも熱が発生しにくくなる
- 過剰反応を抑える

#### オキシトシン的パラメータ `o_t`
- 自己境界を拡張する
- 対象を自己に近い重みで扱う
- 関係性の重みづけを大きく変える

#### ノルアドレナリン的パラメータ `n_t`
- 危険性の高い誤差を増幅する
- 平時モードから非常時モードへの切り替えを促進する
- 制御不能感や脅威感を強める

---

## 5. 関係性パラメータ

人間モデルでは、入力対象との関係強度を別途持つ。

```math
r_t \in [0,1]
```

- `r_t = 0` : ほぼ無関係・外部対象
- `r_t = 1` : 強い結合対象・拡張自己に近い対象

この `r_t` は以下のようなものを含みうる。

- 信頼
- 親密性
- 愛着
- 共同体的所属
- 共有前提の厚さ

---

## 6. 重み行列

軽量版では、重み行列 `W_t` は対角行列として扱う。

```math
W_t = \mathrm{diag}(w_1, w_2, ..., w_m)
```

各重みは基層構造と関係性から生成する。

```math
w_i = \alpha_i + \beta_i n_t - \gamma_i s_t + \delta_i o_t r_i
```

- `\alpha_i` : 基本感度
- `\beta_i` : 危機増幅係数
- `\gamma_i` : 誤差緩和係数
- `\delta_i` : 関係拡張係数
- `r_i` : 入力対象 `i` への関係強度

### 解釈
- `n_t` が高いほど、危険誤差は熱になりやすい
- `s_t` が高いほど、誤差は熱化しにくい
- `o_t r_i` が高いほど、関係対象の誤差は重く感じられる

---

## 7. 微修正モデル

熱は、微修正を通じて構造へ変換される。

```math
A_{t+1} = A_t + \eta_t \Delta A_t
```

ここで、更新方向は熱を減少させる方向として与える。

```math
\Delta A_t = - \nabla_A H_t
```

- `\Delta A_t` : 微小更新方向
- `\eta_t` : 学習率

### 解釈
- 整合とは、熱を構造へ変換する過程である
- 学習とは、微修正の累積である
- 成長とは、微修正の軌跡である

---

## 8. 学習率

学習率 `\eta_t` は固定値ではなく、人間モジュール側の状態に依存させる。

```math
\eta_t = \eta_0 (1 + \lambda d_t)(1 + \mu c_t)
```

- `\eta_0` : 基本学習率
- `d_t` : 探索性
- `c_t` : 意識的観測・内省強度
- `\lambda, \mu` : 係数

### 解釈
- 探索性が高いほど更新しやすい
- 意識的観測が働くほど微修正が入りやすい

---

## 9. 整合慣性モデル

整合によって解消・再配置された熱は、整合慣性の材料になる。

```math
I_{t+1} = \rho I_t + \kappa R_t
```

```math
R_t = \max(0, H_t - H_{t+1})
```

- `I_t` : 整合慣性
- `R_t` : 今回の整合によってどれだけ熱が減少したか
- `\rho` : 慣性保持率
- `\kappa` : 固定化係数

### 解釈
- 熱はそのまま慣性になるのではない
- 整合された熱が構造に固定され、慣性になる
- 慣性は「過去に整合された熱の記録」である

---

## 10. 跳躍条件

熱が大きすぎる場合、微修正では吸収できず、非連続更新が起きる。

```math
\text{if } H_t \ge \Theta_t, \quad A_{t+1} = A_t + \Delta A_t^{(L)}
```

- `\Theta_t` : 跳躍閾値
- `\Delta A_t^{(L)}` : 大規模更新

閾値自体も人間状態に依存させる。

```math
\Theta_t = \Theta_0 + a s_t - b n_t + c g_t
```

- `\Theta_0` : 基本閾値
- `g_t` : 関係的安全性・信頼基盤
- `a, b, c` : 係数

### 解釈
- セロトニンが高いと耐えやすい
- ノルアドレナリンが高いと跳躍しやすい
- 信頼基盤が高いと柔軟な微修正で済みやすい

---

## 11. 感情出力モデル

感情は、熱処理の出力形式として扱う。

```math
y_t = F(H_t, I_t, r_t, p_t, \sigma_t)
```

- `y_t` : 感情出力
- `\sigma_t` : 主観的制御可能感

軽量版では例として以下を置く。

### 怒り
```math
\mathrm{anger}_t \propto H_t (1-r_t) n_t
```

- 高熱
- 関係性が低い
- 危機増幅が高い

ときに出やすい。

### 恐怖
```math
\mathrm{fear}_t \propto H_t (1-r_t)(1-\sigma_t)n_t
```

- 高熱
- 関係性が低い
- 制御不能感が高い
- 危機増幅が高い

ときに出やすい。

### ケア・保護
```math
\mathrm{care}_t \propto H_t r_t o_t
```

- 高熱
- 関係性が高い
- 境界拡張が高い

ときに出やすい。

### 解釈
感情とは、誤差という熱に対し、関係性と基層構造によって決定される出力形式である。

---

## 12. 意識-無意識インターフェース

本仕様では、意識・無意識・学習層を以下のように分離する。

### 無意識層
```math
x_{t+1} = A_t x_t
```

- 高速・並列処理
- 既存の整合慣性そのもの
- 実質的な意思決定主体

### 意識層
```math
H_t = \|W_t e_t\|
```

- 無意識の出力を観測
- 違和感・比較・誤差評価を行う
- 熱の検出器として機能する

### 学習層
```math
A_{t+1} = A_t + \eta_t \Delta A_t
```

- 意識的観測結果に基づき微修正を行う
- 微修正の蓄積により無意識構造を書き換える

### 解釈
- 無意識 = 慣性
- 意識 = 熱検出と評価
- 学習 = 熱の構造化

---

## 13. 行動解釈の基本指針

### 13.1 小さい誤差
- 熱は小さい
- 微修正は小さい
- 習慣強化や微学習が起きる

### 13.2 適度な誤差
- 熱は十分に感じられる
- 整合できれば報酬が高い
- 整合慣性が強く形成される

### 13.3 大きすぎる誤差
- 熱が処理能力を超える
- 微修正不能
- 跳躍、拒絶、回避、防御反応が起きやすい

---

## 14. 関係性に関する補助解釈

### 14.1 信頼
信頼は、誤差を脅威ではなく変形可能な情報として扱える状態である。

### 14.2 愛
愛は、対象を自己境界内へ取り込み、誤差を外部問題ではなく内部問題として処理する強結合状態である。

### 14.3 裏切り
裏切りは、誤差そのものではなく、誤差処理基盤である信頼への攻撃として機能する。

### 14.4 怒り
怒りは、再整合が困難な状況において、現行構造を維持するために誤差を外部へ転写・排出する熱散逸機構である。

## 16. 最小実装セット

最小限の軽量構成だけ抜き出すなら以下でよい。

```math
x_{t+1} = A_t x_t
```

```math
e_t = x_{t+1} - \hat{x}_{t+1}
```

```math
H_t = \|W_t e_t\|
```

```math
A_{t+1} = A_t - \eta_t \nabla_A H_t
```

```math
I_{t+1} = \rho I_t + \kappa (H_t - H_{t+1})_+
```

この最小形に対し、必要に応じて以下を追加する。

- 関係性 `r_t`
- 基層パラメータ `p_t`
- 意識パラメータ `c_t`
- 制御可能感 `\sigma_t`
- 跳躍閾値 `\Theta_t`
