# RDFエージェントの流体移動アルゴリズム

## 概要

本モジュールは、RDFエージェントの移動を「目的地への直進」ではなく、**複数の素流圧を受けた構造が局所整合方向へ流れる更新過程**として記述するための設計文書である。

従来の移動モデルでは、

- 目標点を設定する
- その点に向かって進む
- 障害物に当たったら対処する

という、点目標追跡型の発想が主であった。

しかしRDF的には、移動もまた構造更新の一形態であり、
エージェントは世界から受ける多様な素流圧の合成の中で、
その瞬間ごとの整合しやすい方向へ流れていくものと捉える方が自然である。

したがって本アルゴリズムでは、移動を以下のように再定義する。

> 移動とは、エージェントが食料・水・危険・地形・視覚予測・未知性などの素流圧を受け、その局所的な整合方向へ連続的に更新される過程である。

---

## 基本発想

### 1. エージェントは点目標を追うのではない

エージェントは静的な到達点に向かうのではなく、
その場所・その時点における局所圧場の勾配に沿って流れる。

したがって移動の基本単位は、

- 「どこへ行くか」

ではなく、

- 「次の瞬間にどの方向がもっとも整合しやすいか」

である。

### 2. 障害物は壁ではなく圧である

壁・川・深水・狭所などは、単なる衝突判定対象ではなく、
移動場を歪める構造として扱う。

- 近い壁は強い反発圧となる
- 狭い場所は高コスト領域となる
- 視界不良領域は不確実性圧を持つ
- 群れや他個体も局所的な動的障害になりうる

つまり「通れない」以前に、
**行きづらい圧場**として移動更新に影響する。

### 3. 視覚は未来衝突の先取り機能である

視覚や感覚器があるなら、壁は接触時に初めて発見されるのではなく、
事前に進路を歪める要素として認識される。

したがって視覚とは、

> 未来の衝突を、現在の圧として先取りする機能

と解釈できる。

この考え方により、移動は「ぶつかってから回避」ではなく、
**当たる前に流れが変形する**ものとなる。

---

## RDF的対応関係

- エージェント個体：構造
- 周囲の食料・水・敵・壁・群れ：素流圧源
- 行動状態：局所整合モード
- 移動方向：局所整合方向
- スタック：局所圧場設計の不備、あるいは偽の安定井戸
- escape処理：粗い跳躍近似

---

## 素流圧の構成

エージェントが受ける移動圧は、少なくとも以下のような成分に分解できる。

### 1. 目的圧

- 食料への引力
- 水への引力
- 繁殖・群れ・休息などへの引力

### 2. 危険圧

- 捕食者からの反発
- 既知危険地帯からの反発
- 追跡線上から外れようとする圧

### 3. 地形圧

- 壁・深水・段差などの反発
- 森やぬかるみなどの移動コスト
- 狭所や袋小路の圧縮コスト

### 4. 視覚予測圧

- 前方閉塞の先行検知
- 未来衝突の予測反発
- 開けた方向への優先性

### 5. 未知性圧

- 草むら・暗所・未探索領域への不確実性コスト
- ただし状況に応じて符号反転しうる

### 6. 慣性圧

- 急旋回の抑制
- すでに流れている方向の維持

### 7. ノイズ / 揺らぎ

- 微小探索
- 停滞防止
- 局所最適からの離脱余地

## 移動の基本式（概念形）

移動方向ベクトルを以下のように考える。

```text
V_move = w_food F_food
       + w_water F_water
       + w_danger F_danger
       + w_terrain F_terrain
       + w_vision F_vision
       + w_unknown F_unknown
       + w_inertia F_inertia
       + ξ
```

ここで：

- `F_x` は各圧成分の局所ベクトル
- `w_x` は状況依存の重み
- `ξ` は微小揺らぎ

最終移動は `V_move` の方向へ、
速度・疲労・地形コストを考慮して短距離更新する。

---

## 近傍セル評価方式（v4段階）

連続圧場への完全移行前の中間段階として、
近傍セル評価方式を採用できる。

### 手順

1. 周囲8方向または16方向の候補セルを取得
2. 各セルについて以下を評価
   - 食料への近さ
   - 水への近さ
   - 危険からの遠さ
   - 地形コスト
   - 壁近接ペナルティ
   - 慣性ボーナス
   - 微小ノイズ
3. 合計スコア最大の方向を選択
4. 速度ベクトルへブレンドして更新

### 位置づけ

これはまだ離散的だが、
「点目標追跡」よりもはるかにRDF的であり、
**局所圧の合成による移動**の近似として有効である。

---

## 未知領域の符号反転

未知領域は常に障害とは限らない。

たとえば草食個体にとって草むらは通常、

- 視界不良
- 不確実
- 捕食リスク不明

という理由で進入コストを持つ。

しかし、後方に明確な危険がある場合、
その未知領域はむしろ

- 遮蔽となる可能性
- 追跡を切れる可能性
- 既知危険より相対的にましな経路

として評価が反転しうる。

したがって未知性圧は固定マイナスではなく、
**危険圧や恐怖状態に応じて重みが変わる動的項**とする。

概念的には：

```text
平時:      w_unknown < 0
危険時:    w_unknown → 0
極限逃走時: w_unknown > 0
```

つまり未知領域は、

> 障害物ではなく、条件付き経路である

と捉える。

---

## スタックの再定義

従来のスタックは、
「目的地に向かったが進めなかった状態」と見なされていた。

しかしRDF流体移動では、スタックは

> 局所圧場の設計が悪く、偽の安定井戸や渦が形成されている状態

と解釈できる。

したがって根本対処は、

- escapeを強くすること

ではなく、

- 圧場を滑らかにする
- 視覚先読みを入れる
- 行き止まりや狭所のコストを先に乗せる
- 局所失敗方向への短期記憶を加える

ことにある。

escape処理は最後の保険としてのみ残すのが望ましい。

## 実装段階案

### v4
- 近傍8方向採点
- `tx, ty` 主体の直進を廃止
- 地形コストと壁近接ペナルティ導入
- escapeを保険化

### v5
- 視覚扇形による前方壁圧
- 未知領域の条件付き評価反転
- 失敗方向の短期記憶
- 混雑圧 / 群れ圧の導入

### v6
- 近傍セル採点を超えて、連続ベクトル場合成へ移行
- セルではなく局所勾配ベースの移動
- 個体群全体を動的圧場として扱う

---

## ノルアドレナリンスイッチと素流圧再編

本アルゴリズムでは、危険検知時のモード切替を単なる「恐怖値の増加」として扱わない。
むしろ、**ノルアドレナリンスイッチによって世界の素流圧の解釈そのものが再編される**と考える。

平時においては、

- 草 = 食料源
- 水辺 = 飲水地点
- 開けた場所 = 移動しやすい領域
- 草むら = 不確実性の高い未知領域

として読まれていたものが、危険検知後には以下のように反転しうる。

- 草 = 食料源 → 遮蔽候補
- 草むら = 未知領域 → 隠れる場所 / 追跡遮断候補
- 開けた場所 = 移動しやすい領域 → 露出リスク領域
- 水辺 = 飲水地点 → 移動阻害や逃走制限の可能性

これは、対象そのものが変わるのではなく、
**構造側の優先整合モードが変わることで、同じ対象の素流圧ラベルが変化する**ことを意味する。

したがって非常時モードでは、単に危険圧を強めるだけでなく、
以下のような評価項の再構成が必要になる。

- 食料圧の重み低下
- 飲水圧の重み低下
- 遮蔽圧の新規付与または増大
- 開放地への露出リスク圧の付与
- 逃走経路確保圧の立ち上げ

概念的には以下のように表せる。

```text
平時:
V_move = F_food + F_water + F_unknown(-) + F_inertia + ξ

危険時:
V_move = F_danger + F_cover + F_escapeRoute + F_openRisk(-) + ξ
```

ここで重要なのは、危険時は単なる重み変更だけでなく、
**世界の素流圧マップそのものの意味座標が切り替わる**ことである。

これはRDFエージェントにおいて、
ノルアドレナリンを「危険信号増幅物質」としてだけでなく、

> 世界解釈モードを平時から非常時へ切り替える基層スイッチ

として位置づけることを意味する。

そのため、流体移動アルゴリズムにおけるノルアドレナリンスイッチは、
行動モード変更ではなく、**素流圧ラベル再編モジュール**として実装されるのが望ましい。

## まとめ

RDFエージェントの移動は、
経路探索問題としてではなく、
**局所圧場の中で構造が整合方向へ流れる更新現象**として捉えるのが自然である。

このとき、

- 壁は衝突対象ではなく反発圧場
- 視覚は未来衝突の現在化
- 未知領域は条件付き経路
- スタックは圧場設計不備

として再解釈される。

したがって理想的な移動アルゴリズムは、

> 点目標へ進むAIではなく、複数の素流圧の合成の中で連続的に流れるAI

である。

これはRDFにおける「構造」「素流圧」「整合」「跳躍」の発想を、
エージェント移動へ直接接続する基礎モジュールといえる。
