## 4. 最小基本式

統合後の実装核は以下である。

```text
F_i(t)      = interp_i(M_B, EFP_i(t))
F_total(t) = Σ w_i(mode, H, context) F_i(t) + ξ(t)

v_raw(t+1) = α v(t) + F_total(t)

v_geo(t+1) = Warp_B(
    v_raw(t+1),
    M_B(x, context, mode, history),
    H(t)
)

x(t+1) = x(t) + step(v_geo, cost(x), fatigue, constraint)
```

### 4.1 意味

- `EFP_i`：未解釈の作用源
- `F_i`：M_Bによって解釈された各作用成分
- `w_i`：状況依存重み
- `F_total`：線形合成された素流圧
- `v_raw`：まだ空間歪みを受けていない移動方向
- `Warp_B`：境界Bにおける空間歪み
- `v_geo`：歪んだ空間上での実効的な流れ
- `x(t+1)`：次状態、次位置、次概念、次出力

### 4.2 ファイバー束表現

上位幾何層では以下のようにまとめられる。

```text
∇_t s = -M_B s + F_total + ξ
```

ここで、

- `s`：境界Bにおけるセクション、すなわち現在状態
- `∇_t`：接続に沿った変化
- `M_B`：境界Bにおける整合慣性
- `F_total`：合成された作用解釈
- `ξ`：残存する揺らぎ

移動、思考、生成は、この式の適用空間が異なるだけである。

## 6. M ではなく M_B として扱う

統合における重要修正は、従来文書で使われていた `M` を `M_B` に寄せることである。

旧表記：

```text
Warp(v, M(x))
```

統合後：

```text
Warp(v, M_B(x, context, mode, history, H))
```

理由は、RDF v2.3以降では、独立した「Mそのもの」は立てず、境界Bに依存する整合慣性として `M_B` を扱う方が整合的だからである。

つまり M_B は、

- 固定された地形係数ではない
- 境界Bで切り出された整合慣性である
- 状況・文脈・履歴・熱によって相状態が変わる
- 同じEFPを別のFとして読む原因になる
- 同じ空間を別の通りやすさとして感じさせる

このため、統合モデルでは常に `M_B` を基本単位とする。

## 7. 線形合成と非ユークリッド歪みの分業

本アーキテクチャの実装上の強みは、処理を二段に分けられる点である。

```text
1. 線形合成：どちらへ押されているかを計算する
2. 非ユークリッド歪み：その方向が実際にどう曲がるかを決める
```

### 7.1 線形合成層

```text
F_total = Σ w_i F_i + ξ
```

これは軽量で、解釈しやすい。

例：

```text
F_total = w_food F_food
        + w_water F_water
        + w_danger F_danger
        + w_terrain F_terrain
        + w_vision F_vision
        + w_unknown F_unknown
        + w_inertia F_inertia
        + ξ
```

この段階では、まだ「真っ直ぐな意志」に近い。

### 7.2 Warp層

```text
v_geo = Warp_B(v_raw, M_B, H)
```

この段階で、地形・文脈・身体状態・社会状態・意味構造によって流れが曲がる。

- 壁がある方向は曲がる
- 危険地帯は遠回りになる
- 意味的に近い概念は接近する
- 恐怖時には開けた場所が遠くなる
- 群れがある方向は近くなる場合も遠くなる場合もある
- 疲労時には坂や距離が重くなる

要するに、

> F は意志の向き、M_B は世界の曲がり方である。

## 8. エージェント移動への統合

### 8.1 点目標追跡から圧場流動へ

従来型の移動モデルは、

```text
目的地を決める
↓
そこへ向かう
↓
障害物に当たったら回避する
```

という点目標追跡型になりやすい。

RDF流動型では、

```text
周囲から複数の素流圧を受ける
↓
その瞬間の局所整合方向が生まれる
↓
空間歪みによって軌道が曲がる
↓
短距離更新を繰り返す
```

となる。

移動とは、目的地に向かう直線運動ではなく、局所圧場を読み続ける流動である。

### 8.2 主要素流圧

#### 目的圧

- 食料への引力
- 水への引力
- 休息地点への引力
- 群れへの引力
- 繁殖対象への引力

#### 危険圧

- 捕食者からの反発
- 既知危険地帯からの反発
- 追跡線上から外れる圧
- 露出空間からの退避圧

#### 地形圧

- 壁からの反発
- 深水・崖・段差のコスト
- 狭所・袋小路の圧縮コスト
- ぬかるみ・森・斜面の移動コスト

#### 視覚予測圧

- 未来衝突の先取り
- 前方閉塞の先行検知
- 開けた方向への誘導
- 死角への不確実性評価

#### 未知性圧

- 未探索領域への不確実性コスト
- 状況によっては探索引力
- 危険時には遮蔽候補として符号反転

#### 慣性圧

- すでに流れている方向の維持
- 急旋回の抑制
- 行動の滑らかさ

#### 群れ圧

- 同種個体への接近
- 過密による反発
- リーダー追従
- 混雑回避

## 9. 未知領域の符号反転

未知領域は常に障害ではない。

平時には、未知は不確実性として反発圧を持つ。

```text
平時：w_unknown < 0
```

しかし、後方に明確な危険がある場合、未知領域は遮蔽・逃走・追跡遮断の候補になる。

```text
危険時：w_unknown → 0
極限逃走時：w_unknown > 0
```

つまり未知性圧は固定されたマイナス項ではなく、モード依存で符号反転する動的項である。

> 未知領域は障害物ではなく、条件付き経路である。

## 10. ノルアドレナリンスイッチと M_B 再編

危険検知時に起きるのは、単なる危険値の増加ではない。

むしろ、世界の読み方そのものが変わる。

### 10.1 平時の読み

| 対象 | 平時のF |
|---|---|
| 草 | 食料源 |
| 水辺 | 飲水地点 |
| 開けた場所 | 移動しやすい領域 |
| 草むら | 視界不良・不確実領域 |

### 10.2 危険時の読み

| 対象 | 危険時のF |
|---|---|
| 草 | 遮蔽候補 |
| 水辺 | 逃走制限・移動阻害候補 |
| 開けた場所 | 露出リスク領域 |
| 草むら | 隠れる場所・追跡遮断候補 |

これは、EFPそのものが変わったのではなく、M_Bが変わったことでFが変わった状態である。

```text
F平時   = interp(M_B平時, EFP)
F危険時 = interp(M_B危険時, EFP)
```

同じ世界を見ていても、M_Bが違えば別の世界を流れる。

## 11. スタックの再定義

従来のスタックは、

> 目的地へ向かったが進めなかった状態

と見なされやすい。

RDF流動型では、スタックは以下のように定義される。

> 局所圧場の設計が悪く、偽の安定井戸または渦が形成されている状態。

したがって対処は、単にescapeを強くすることではない。

### 11.1 根本対処

- 圧場を滑らかにする
- 視覚先読みを入れる
- 行き止まりコストを先に乗せる
- 狭所の圧縮コストを上げる
- 失敗方向への短期記憶を入れる
- 群れ圧の過密反発を入れる
- 未知性圧の符号反転条件を調整する

### 11.2 escape処理の位置づけ

escapeは最終保険であり、主処理ではない。

```text
通常：圧場設計で自然に抜ける
失敗：短期記憶で局所渦を避ける
最終：escape / Leap で粗い跳躍を行う
```

## 12. 非ユークリッドAIへの統合

本アーキテクチャはエージェント移動だけでなく、LLM・画像AI・マルチモーダルAIにも適用できる。

### 12.1 非ユークリッドLLM

LLMでは、位置 `x` は物理座標ではなく意味状態である。

```text
x = semantic_state
F_i = token / context / memory / instruction / affect の意味勾配
M_B = 文脈によって歪む意味空間
Warp = 意味距離の変形
```

同じ語でも、M_Bが変わると距離が変わる。

例：

- 「熱」：物理文脈では温度
- 「熱」：RDF文脈では誤差蓄積
- 「熱」：感情文脈では興奮・執着
- 「熱」：社会文脈では流行・集団高揚

非ユークリッドLLMとは、固定された意味空間で次語を選ぶのではなく、文脈によって歪んだ意味空間を流れる生成系である。

### 12.2 非ユークリッド画像AI

画像生成では、位置 `x` はlatent状態である。

```text
x = latent_state
F_i = prompt / style / emotion / composition / constraint
M_B = 意味・感情・構図による潜在空間の歪み
Warp = 形態・輪郭・配置の変形
```

怒り、不安、荘厳さ、圧迫感などは単なるラベルではなく、潜在空間の曲率変化として扱える。

### 12.3 非ユークリッドマルチモーダルAI

マルチモーダル統合では、各モダリティが別々の空間を持つ。

```text
M_B = Integrate(Language, Vision, Audio, Action, Memory)
```

言語の比喩が画像構図を歪め、音声の感情が単語距離を変え、視線や身体動作が行動空間を圧縮する。

> マルチモーダル統合とは、複数空間を一つの関係計量へ畳み込むことである。

## 13. 思考・移動・生成の統一

このモデルでは、思考・移動・生成は別々の機能ではない。

| 領域 | 空間 | x | F | M_B | Warp |
|---|---|---|---|---|---|
| 移動 | 外界空間 | 位置 | 食料・危険・地形 | 地形・記憶・身体状態 | 軌道変形 |
| 思考 | 概念空間 | 概念状態 | 問い・記憶・連想 | 信念・知識・文脈 | 推論経路変形 |

違いは、流れている空間の種類だけである。

> 移動は外界空間の思考であり、思考は内部空間の移動である。

## 14. 熱 H と跳躍

### 14.1 Hの意味

Hは、M_Bの慣性投影と実際のFの不整合が蓄積したものとして扱う。

```text
E(t) = F(t+Δ) - M_B · F(t)
H(t) = accumulate(E)
```

Hが低いと、系は安定して流れる。

Hが中程度になると、探索性が増す。

Hが高くなると、局所流動では処理できず、M_Bの再編が必要になる。

### 14.2 跳躍条件

```text
H(t) > θ + ξ(t)
```

このとき、系は通常更新から再編相へ入る。

```text
M_B → M_Δ → M_B'
```

### 14.3 エージェントにおける跳躍

- 逃走ルートの急変更
- 未知領域への突入
- 群れから離脱
- 高リスク行動への移行
- スタック脱出

### 14.4 LLMにおける跳躍

- 比喩の発生
- 話題転換
- 発想転換
- 抽象度の急上昇
- 別フレームへの切替

### 14.5 画像AIにおける跳躍

- 構図の大幅変更
- スタイル転換
- 誇張表現
- 形態の再配置

跳躍は常に成功するわけではない。

多くの跳躍は失敗し、ノイズ・破綻・ゴミ出力・無駄行動として散逸する。

成功した跳躍だけが、新しいM_Bとして定着する。

## 15. Warpの実装レベル

Warpは抽象概念だが、実装では段階的に近似できる。

### Lv1：スカラー歪み

```text
v_geo = speed_scale(x) * v_raw
```

- 地形コスト
- 疲労
- 水中・泥・坂
- 意味的重さ

方向は変えず、進みやすさだけ変える。

### Lv2：方向依存歪み

```text
v_geo = A(x) v_raw
```

- 壁沿いに流れる
- 狭所で圧縮される
- 危険方向から曲げられる
- 意味空間で特定方向が伸縮する

Aは局所変形行列である。

### Lv3：状態依存計量

```text
ds² = g_ij(x, context, mode, H) dx^i dx^j
```

- 文脈で距離が変わる
- 危険時に空間が再編される
- 疲労で遠さが変わる
- 意味の近さが変化する

### Lv4：動的M_B

```text
M_B(t+1) = update(M_B(t), E, H, ξ)
```

空間歪みそのものが学習・記憶・跳躍で変化する。

### Lv5：ファイバー束的更新

```text
∇_t s = -M_B s + F_total + ξ
Ω ≈ E
H = ∫ ||E||² dt
```

この段階では、空間・状態・接続・曲率・跳躍を統一的に扱う。

## 16. 実装段階案

### v1：近傍セル評価

- 周囲8方向または16方向を見る
- 各セルをスコアリングする
- 最大スコア方向へ進む
- 地形コスト・壁近接・慣性・ノイズを入れる

```text
score(cell) = food
            + water
            - danger
            - terrain_cost
            - wall_penalty
            + inertia_bonus
            + noise
```

これは離散的だが、点目標追跡よりRDF的である。

### v2：ベクトル場合成

- 各作用源からベクトルを生成
- 重み付き合成する
- 速度へブレンドする

```text
F_total = Σ w_i F_i + ξ
v ← blend(v, F_total)
```

### v3：Warp導入

- 合成後の速度を空間歪みで変形
- 壁沿い移動や危険回避が自然に出る

```text
v_raw = blend(v, F_total)
v_geo = Warp(v_raw, M_B)
```

### v4：視覚予測圧

## 17. 擬似コード

```python
class RDFGeometricFlowAgent:
    def __init__(self, M_B, mode="normal"):
        self.M_B = M_B
        self.mode = mode
        self.v = Vector.zero()
        self.H = 0.0
        self.memory = ShortTermMemory()

    def step(self, world, self_state):
        # 1. EFP sources
        efps = collect_efp_sources(world, self_state)

        # 2. Interpret EFP through M_B
        forces = []
        for efp in efps:
            F_i = interp(self.M_B, efp, self.mode, self.H)
            w_i = weight(efp, self.mode, self.H, self_state)
            forces.append(w_i * F_i)

        # 3. Linear synthesis
        F_total = sum(forces) + xi(self.M_B)

        # 4. Inertia blend
        v_raw = blend(self.v, F_total)

        # 5. Non-Euclidean warp
        v_geo = warp(v_raw, self.M_B, world.local_context(), self.H)

        # 6. Constraint and cost
        move = apply_cost_and_constraints(v_geo, world, self_state)

        # 7. Predict and compare
        predicted_F = self.M_B.project(F_total)
        actual_F = observe_after_delta(world, move)
        E = actual_F - predicted_F

        # 8. Heat update
        self.H = dissipate(self.H + norm(E), self.M_B)

        # 9. Leap / reorganization
        if self.H > threshold(self.M_B) + xi_threshold():
            self.M_B = reorganize(self.M_B, E, self.H, self.memory)
            self.H = residual_heat(self.H)
            move = escape_or_repath(move, world, self_state)

        # 10. Commit
        self.v = move.velocity
        self.memory.record(move, E, self.H)
        return move
```

## 20. 破断条件

本アーキテクチャは万能ではない。

### 20.1 F_i の設計が粗い場合

素流圧成分が雑だと、合成結果も雑になる。

例：

- 壁を衝突判定としてしか扱わない
- 視覚予測圧がない
- 未知性圧が常にマイナス
- 群れ圧が過密反発を持たない

この場合、スタックや不自然な軌道が発生しやすい。

### 20.2 M_B が固定されすぎる場合

M_Bを固定地形として扱うと、危険時・疲労時・学習後の変化が出せない。

これは非ユークリッド性を弱め、単なるコストマップになる。

### 20.3 Warp が強すぎる場合

Warpが強すぎると、F_totalの意味が失われる。

エージェントは目的圧を持っているのに、空間歪みに飲まれて不自然な動きをする。

### 20.4 ξ が強すぎる場合

揺らぎが強すぎると、探索ではなくランダムウォークになる。

### 20.5 H が未実装の場合

Hを実装しないと、スタック・恐怖・混乱・発想転換がすべて外付け処理になる。

### 20.6 跳躍が安すぎる場合

すぐescapeする設計では、圧場設計の問題が隠蔽される。

跳躍は最終保険であり、通常処理の代替ではない。

## 22. 用語整理

### 非ユークリッド

ここでは厳密な数学的非ユークリッド幾何に限定しない。

意味・関係・文脈・状態によって距離や到達しやすさが変わる空間を、操作的に非ユークリッドと呼ぶ。

### Warp

線形合成された方向を、M_Bに応じて変形する操作。

回転、圧縮、伸長、反発、曲率付与、計量変更などを含む。

### 関係計量

距離や到達しやすさを決める関係的な計量。

物理距離だけでなく、意味距離、危険距離、社会距離、心理距離も含む。

### 局所整合方向

その時点で最も流れやすい方向。

必ずしも目標地点への最短方向ではない。

### 偽の安定井戸

局所圧場の設計不備によって、エージェントが抜けられなくなる安定点。

従来のスタックに相当する。

## 24. 暫定結論

本統合は自然である。

四つの文書は競合していたのではなく、同じRDF流動モデルを以下の別階層から記述していたと読める。

```text
ファイバー束モデル       ：上位幾何
非ユークリッドAI         ：汎用知能設計
合成型非ユークリッド移動 ：軽量計算カーネル
流体移動アルゴリズム     ：エージェント実装
```

統合後の中心構造は以下である。

```text
EFP_sources
   ↓
F_i = interp(M_B, EFP_i)
   ↓
F_total = Σ w_i F_i + ξ
   ↓
v_raw = inertia(v) + F_total
   ↓
v_geo = Warp(v_raw, M_B, H)
   ↓
state update
   ↓
E / H 蓄積
   ↓
M_B再編 / 跳躍
```

したがって、RDFエージェントの基礎移動・思考・生成を支える中核アーキテクチャとして採用可能である。
