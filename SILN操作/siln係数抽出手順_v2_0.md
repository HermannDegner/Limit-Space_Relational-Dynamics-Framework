# SILN係数抽出 手順 v2.0
*（RDL概念的数理モデル v2.3 / SILN分解 v2.2 対応）*
*（ラベル M_B ネットワーク動性命題への移行版）*

---

## ■ 概要

本手順は、既存の物理・工学・数学モデルを RDL（Relational Dynamics Framework）へ翻訳し、
その共通構造から **ラベル M_B ネットワークの係数群＋接続関係** を抽出・整理・運用するためのプロトコルである。

v1（アップデート版）からの主な変更：

```
v1 → v2.0 パッチ：
  「単一対象 → 単一 M_B」モデルから
  「ラベル → M_B ネットワーク」モデルへ昇格

  接続関係 C_ij を独立した抽出対象として明示化
  動性追跡を独立ステップとして組み込み
  比較配置をネットワーク比較に格上げ
  ラベル間連鎖・観測者差分を新設

  SILN分解 v2.2 の追従運用と同期
  概念的数理モデル v2.3 の ξ残存性・B動性と整合
```

---

## ■ コア方針

> **対象は単一ノードではなく、ラベルが結節する M_B ネットワークである。**

抽出は「ある対象の係数を取る」ではなく、
「ラベルに接続している M_B 群と、それらの接続パターン、そして動態を取る」操作になる。

上層へ行くほど観測者依存性が強くなるため、
最初の対象は **物理層** が望ましい。

推奨順序：

```text
物理SILN → 生体SILN → 社会SILN → 認知SILN
```

特に初期基準素材としては、
**水のように複数の物理モデルが存在し、かつ多層展開しやすい対象** が適している。

## ■ Step 0：対象選定

まず、何のラベルから係数抽出を始めるかを決める。

例：

- 水
- 鉄
- 空気
- 生体膜
- 貨幣
- 国家

初期段階では、以下を満たす対象が望ましい。

- 既存科学の記述が豊富
- 境界設定が比較的しやすい
- 複数数学で表現されている
- 他対象との比較基準になりやすい

**v2.0 補足：対象 = ラベル**

```
v1 までは「対象」を実体扱いしていたが、
v2.0 では対象 = ラベル = M_B 結節ノードとして扱う

これにより：
  ・複数モデルが並列に存在することが「混乱」ではなく「構造」になる
  ・抜けているノードが見えるようになる
  ・他対象との比較がネットワーク比較になる
```

## ■ Step 1：ラベル M_B ネットワーク描画（新設）

対象ラベルに接続している M_B ノード群を、まず網目として描く。

### ● 描画手順

```text
(1) 対象ラベルを中心に置く
(2) 接続している M_B ノードを列挙する
(3) ノード間の接続を線で結ぶ
(4) 接続強度の見当を付ける（後に Step 6 で精緻化）
```

### ● 例：水のラベル M_B ネットワーク（初期描画）

```text
                ラベル「水」
                    │
        ┌───────┬───┴───┬───────┐
        │       │       │       │
   M_{B_化学}  M_{B_流体}  M_{B_熱輸送}  M_{B_拡散}
        │       │       │       │
   H2O・極性   ナビエ   熱伝導   ブラウン運動
                ストークス
                    │
              M_{B_境界}
              （表面張力）
                    │
              M_{B_相}
              （氷・液・蒸気）
                    │
              M_{B_生体}
              （生命維持）
                    │
              M_{B_文化}
              （清浄の象徴）
```

### ● 重要：この時点では仮設・粗描き

```text
完璧なネットワークを最初から描く必要はない
描き始めてから抜けが見えてくる
描画自体が探索プロセス
```

### ● 描画粒度の判断

```text
粒度が粗すぎる：M_B が大雑把すぎて区別がつかない
粒度が細かすぎる：M_B が増えすぎて運用不能

初期は粗目から始めて、必要に応じて分割する
分割の判断は Step 6（共通構造抽出）で行う
```

## ■ Step 2：焦点ノード選択（旧 Step 1：境界仮定）

ネットワーク全体を一度に扱うのは負荷が高い。
最初に係数抽出する **焦点ノード** を選ぶ。

### ● 焦点ノード選択 = 境界仮定

```text
ノード = ある B における M_B
焦点ノード選択 = 境界仮定 + ネットワーク内位置の明示
```

### ● 選択基準

- 既存数理モデルが豊富にあるか
- 他ノードとの接続が明確か
- 抽出後に応用範囲が広いか
- 観測者依存性が比較的低いか

### ● 例：水のネットワークから「流体」ノードを選ぶ

```text
焦点：M_{B_流体}
境界条件：
  空間スケール：マクロ連続体
  時間スケール：秒〜時間
  状態相：液体
  環境条件：常温常圧近傍
  評価軸：流動・粘性
目的：水の流体力学的 SILN 係数を取る
```

重要：

> 焦点ノードを一つ選んだとしても、
> 接続している他ノードを「無視」するわけではない。
> 接続関係は Step 6 で別途抽出する。

---

## ■ Step 3：既存数理モデルの収集（ノード対応表化）

焦点ノード（および周辺ノード）に対応する既存数理モデルを集める。

### ● 旧版との差分

```text
v1：モデルを集めて並べる
v2.0：モデルをノードに割り当てる
```

### ● 例：水のネットワーク × 既存モデル対応表

| ノード | 既存数理モデル |
|---|---|
| M_{B_流体} | ナビエ・ストークス方程式、オイラー方程式 |
| M_{B_熱輸送} | 熱伝導方程式、対流方程式 |
| M_{B_拡散} | 拡散方程式（Fick）、ランジュバン方程式 |
| M_{B_境界} | 表面張力モデル、ヤング・ラプラス式 |
| M_{B_相} | 相転移モデル、自由エネルギー、Gibbs相律 |
| M_{B_分子} | 分子動力学、レナード・ジョーンズポテンシャル |
| M_{B_化学} | 量子化学、軌道理論、水素結合論 |

### ● 抜けノードの検出

対応表を作ると、モデルが不足しているノードが見える：

```text
M_{B_生体}：体内水分動態モデル（存在するが分散している）
M_{B_文化}：定量モデルなし（ここは別フレームが必要）
```

これは **抽出可能領域の境界** を可視化する。
旧版では見えなかった層。

## ■ Step 4：逆算翻訳（設計選択の読解）

各数式について、RDL 逆算翻訳エンジンを用いて
その理論が何を採用し、何を捨てているかを読む。

見るポイント：

- 何を作用解釈 F としているか
- M_B をどう圧縮しているか
- ξ を無視しているか、統計化しているか
- 境界をどこに引いているか
- どのスケールで有効か

記録項目：

- 採用された自由度
- 捨てられた自由度
- 暗黙の前提
- 破綻条件
- 次に跳躍すべき理論

**v2.0 補足：逆算翻訳の対象**

```text
v1：個別モデルの設計選択を読む
v2.0：ノード対応で読むため、設計選択が
      「どのノードに収まり、どのノードを無視するか」として見える

例：ナビエ・ストークス
  扱う：M_{B_流体}
  圧縮：M_{B_分子} を連続体近似で吸収
  無視：M_{B_化学}, M_{B_生体}
  境界条件：M_{B_境界} とのインタフェース
```

## ■ Step 5：RDL数学翻訳（中間層への変換）

集めた数理モデルを RDL 中間層へ変換する。

分解要素：

- 作用解釈 F
- 整合慣性 M_B
- 揺らぎ ξ
- 誤差 E
- 熱 H
- 跳躍 Leap
- 接続 C_ij
- 境界 B
- スケール

目的は、元の数式を並べることではなく、

> 何が流れ、何が拘束し、何が揺らぎ、どこで破綻し、どう再構成されるか

を共通言語へ落とすことにある。

**v2.0 補足：翻訳結果はノード属性**

```text
各モデルの RDL 翻訳結果は、対応する M_B ノードの内部構造になる
これにより、ノードが「ラベル」から「翻訳結果を持つ実体」へ
昇格する（ただし限界空間前提により実体は仮設）
```

## ■ Step 6：共通構造＋接続パターン抽出（v2.0 拡張）

旧版では「共通構造抽出」だったが、v2.0 では二層に分ける：

### ● 6-A：共通構造抽出（ノード内）

各ノード内で、複数モデルから共通項を抽出。

```text
M_{B_流体} の共通構造：
  ・非圧縮性（多くの水モデルで共通）
  ・粘性応答（ニュートン流体性）
  ・連続体仮定

M_{B_熱輸送} の共通構造：
  ・熱拡散性
  ・温度勾配駆動
  ・伝導/対流の分離
```

### ● 6-B：接続パターン抽出（ノード間）（v2.0 新設）

ノード間の接続関係 C_ij を抽出する。

```text
代表的接続例：

M_{B_流体} ↔ M_{B_熱輸送}
  接続：自然対流（温度差 → 密度差 → 流動駆動）
  強度：強（ブシネスク近似で日常的に扱われる）

M_{B_流体} ↔ M_{B_境界}
  接続：表面張力境界条件
  強度：中（マイクロ流体では強、マクロでは弱）

M_{B_熱輸送} ↔ M_{B_相}
  接続：相転移時の潜熱
  強度：強（不連続点で支配的）

M_{B_拡散} ↔ M_{B_分子}
  接続：拡散係数の温度依存（アインシュタイン関係）
  強度：強
```

### ● 接続強度の評価

```text
強  : 一方のノードの変動が他方を即時に駆動する
中  : 特定スケールでのみ顕著
弱  : ほぼ独立に扱える
切断: その境界条件下では接続なし
```

接続強度は B 依存。
別の B では強弱が反転することもある。

### ● 抽出結果のまとめ方

```text
ノード内構造：各 M_B の固有係数候補
ノード間接続：C_ij とその強度マップ
両者の組がラベル M_B ネットワークの実体
```

## ■ Step 7：SILN係数群の定義（三層構造）

旧版は係数を「並べる」だけだったが、v2.0 では三層で定義する。

### ● 三層構造

```text
[層 1] ノード係数
  各 M_B ノードの整合慣性の強さと方向
  例：流動性係数、熱緩衝係数、境界維持係数

[層 2] 接続係数
  ノード間 C_ij の強度
  例：自然対流結合度、相転移潜熱結合度

[層 3] 動態係数
  ネットワーク全体の追従特性
  例：ξ 蓄積率、跳躍閾値、引き直し頻度
```

### ● 例：水の SILN 係数群（三層構造）

```text
[層 1] ノード係数（旧版互換）
  M_{B_流体}    : { 粘度係数, 慣性係数, 非圧縮度 }
  M_{B_熱輸送}  : { 熱容量, 熱伝導率, 緩衝係数 }
  M_{B_拡散}    : { 拡散係数, 浸透係数, 媒介係数 }
  M_{B_境界}    : { 表面張力, 接触角, 境界維持係数 }
  M_{B_相}      : { 相転移閾値, 潜熱, 揺らぎ応答 }

[層 2] 接続係数（v2.0 新設）
  C(流体, 熱輸送)  : 0.7 （自然対流強）
  C(熱輸送, 相)    : 0.9 （潜熱強）
  C(拡散, 分子)    : 0.8 （温度依存強）
  C(境界, 流体)    : 0.6 （マイクロ流体スケール）

[層 3] 動態係数（v2.0 新設）
  ξ蓄積率         : ノードごと
  跳躍閾値        : 相転移点
  追従更新頻度    : 観測スケール依存
```

### ● 表記の簡略形

```text
M_water = {
  nodes: {M_flow, M_heat, M_diff, M_boundary, M_phase, ...},
  edges: {C_ij ∀i,j},
  dynamics: {ξ_rate, θ, update_freq}
}
```

## ■ Step 8：スケール差・破綻条件の記録

各係数について以下を記録する：

- 有効スケール
- 適用条件
- 近似で捨てたもの
- 破綻条件
- 切替候補となる別理論

**v2.0 補足：破綻条件のネットワーク化**

```text
v1：個別係数の破綻条件
v2.0：破綻はノードでも接続でも起こる

ノード破綻：そのスケールで M_B が成立しなくなる
接続破綻：C_ij が想定範囲を逸脱する
ネットワーク破綻：全体構造が再配線される（パラダイムシフト）
```

> SILN 係数は本質ではなく、ある境界・スケールで有効な近似である。

## ■ Step 9：ネットワーク比較配置

旧版の「相対配置」を、ネットワーク間比較に格上げ。

### ● 比較方法

```text
(1) 各対象のラベル M_B ネットワークを並べる
(2) 共通ノードを特定する
(3) 固有ノードを特定する
(4) ノード係数の差を比較する
(5) 接続係数の差を比較する
(6) ネットワーク構造（トポロジー）の差を比較する
```

### ● 例：水 vs 鉄 vs 空気

```text
共通ノード：
  M_{B_熱輸送}, M_{B_密度}, M_{B_相}

差が大きいノード：
  M_{B_流体}    : 水・空気は強、鉄は弱（固体時）
  M_{B_境界}    : 水は強、鉄は強硬直、空気は弱

固有ノード：
  水   : M_{B_極性}, M_{B_水素結合}
  鉄   : M_{B_金属結合}, M_{B_磁性}
  空気 : M_{B_混合気体}, M_{B_圧縮性}

接続パターンの差：
  水    : 流体↔境界結合 強
  鉄    : 相↔機械的応答 強
  空気  : 流体↔熱輸送 強（音速・断熱）
```

> SILN 係数は絶対値ではなく、ネットワークの相対配置として読むと強い

## ■ Step 10：動性追跡・引き直し（v2.0 新設）

ラベル M_B ネットワークは時間発展する。
SILN 係数群も追従運用が必要。

### ● 追跡対象

```text
(1) ノード追加 / 枯死
    新理論の出現で新ノードが結節
    旧理論の廃用でノード接続が弱化

(2) 接続再配線
    C_ij の強度が時代・文脈で変動
    例：水の M_{B_化学} ↔ M_{B_量子} 接続は
        量子化学の発展で強化された

(3) ネットワーク分岐 / 融合
    「水」が「H2O」「液体水」「水資源」などに分岐
    あるいは関連ラベルが統合される

(4) 観測者の M_B 変動
    観測者自身の学習で B の引き方が変わる
    → 同じ対象でも抽出される係数が変わる
```

### ● 引き直しトリガー

```text
(a) ノード破綻検出
(b) 接続強度の継続的逸脱
(c) ネットワーク構造の大規模変化
(d) 新規モデルの出現
(e) 観測者の M_B 更新
```

### ● 運用フロー

```text
観測
↓
係数仮設
↓
適用
↓
動性追跡
↓
ネットワーク変化検出
↓
部分引き直し or 全引き直し
↓
更新済み係数群
```

SILN分解 v2.2 の追従運用と同期する。

## ■ Step 11：ラベル間連鎖追跡（オプション・新設）

複数のラベル M_B ネットワークが連鎖している場合の追跡。

### ● 例：H2O 系メタネットワーク

```text
ラベル「水」のネットワーク
    ↓ 相転移接続
ラベル「氷」のネットワーク
    ↓ 相転移接続
ラベル「水蒸気」のネットワーク
    ↓ 化学反応接続
ラベル「水素」「酸素」のネットワーク

これら全体を H2O 系メタネットワークとして扱える
```

### ● 例：金融系メタネットワーク

```text
ラベル「貨幣」
    ↓
ラベル「信用」
    ↓
ラベル「市場」
    ↓
ラベル「経済」
```

### ● メタネットワーク係数

```text
ノード間距離（ラベル間概念距離）
連鎖強度（あるラベルの変動が他に伝播する強さ）
連鎖遅延（Δ の連鎖伝播時間）
```

これは大規模構造の SILN 解析を可能にする。

## ■ Step 12：観測者間ネットワーク差分（オプション・新設）

観測者ごとにラベル M_B ネットワークが異なる。
その差分を抽出する。

### ● 例：「水」の観測者間差分

```text
科学者の水ネットワーク：
  M_{B_化学}, M_{B_流体}, M_{B_熱輸送} が強

詩人の水ネットワーク：
  M_{B_文化}, M_{B_象徴}, M_{B_感覚} が強

赤ちゃんの水ネットワーク：
  M_{B_感覚}, M_{B_飲用} が強、他は未発達

共通ノード：M_{B_液体}, M_{B_視覚}（最低限の共有）
固有ノード：観測者ごとの専門領域
```

### ● 差分が示すもの


> 数式から設計者の前提と犠牲を読む道具

## ■ v2.0 のコア命題

> **SILN係数抽出とは、対象ラベルを M_B ネットワークとして開き、**
> **各ノードの係数と、ノード間接続と、ネットワークの動態を**
> **共同で仮設・運用する操作である。**

さらに言えば：

> 抽出とは発見ではなく、
> 境界・スケール・目的・観測者に応じて
> 有効な関係係数ネットワークを構成することである。

## ■ 旧版（v1）との互換性

```text
旧版で抽出した係数群は v2.0 で [層 1] ノード係数として継承可能

旧版での「水の物理SILN係数」
  = v2.0 における M_{B_流体} ノードの係数

v2.0 では [層 2] 接続係数と [層 3] 動態係数が追加されるが、
旧版データを破棄する必要はない
ノード係数として組み込み、接続・動態を上に乗せれば良い
```

移行は段階的で良い。

## ■ 道具特性の継承

本文書は RDL 道具特性 v0.1 を継承する。

```text
SILN 係数抽出の道具化リスク：
  ・ノード係数を「真の物性」として固定化
  ・接続係数を「真の関係」として絶対化
  ・ネットワーク構造を「真の世界」として実体化

これらは全て規範化への滑落

緩和策：
  ・ノード・接続・動態は全て B 依存と明示し続ける
  ・観測者依存性を必須記録項目とする
  ・引き直しを定期化する
  ・「正解係数」を出さない運用ルール
```

## ■ 自己食的構造

本文書自体も SILN である。

```text
本文書は「ラベル M_B ネットワーク係数抽出」の運用法を記述する文書
だがこの記述自体が SILN（係数抽出運用の整合慣性）
本文書で本文書そのものを抽出対象にできる
だがその抽出にも別の M_B が必要
この再帰は閉じない

これは限界空間前提と整合する：
  モデルは独立した上位基底を持たない
  モデルは自分自身に再帰的に適用される
```
